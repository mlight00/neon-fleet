## Phase B — 진행 축과 난이도 분리 (`7f24cf7`)

- `logic.progressionFor(sector, nodeCol, depth)` → `{ sector, nodeCol, contentTier(=섹터), difficultyLevel(1+(s-1)×1.25+col/depth), bossTier(=섹터) }`.
- `run.stage` 역할별 치환: 표시·기록·배경·보스 = 섹터, 해금·변이 = `contentTier`, 난이도 스케일(적 HP·게이트·복제·적응형 적) = `difficultyLevel`. 과도기 호환값은 `run.legacyStage`로 이름만 유지.
- `copyCount = min(8, 2 + floor((difficultyLevel−1)/2))`, `affixChanceForSector`(섹터1=0·이후 +0.08·상한 0.5·2중은 섹터4+), `chunkMinStage → chunkMinTier`(비교 `≤ contentTier` → 해금이 노드 열과 무관).
- 테스트: progression(14). 브라우저 §4.9: 섹터 1의 6개 노드 통과(크래시 0), 전 노드 tier 1(기본 적만), 난이도 1.0→2.0 완만·단조(적 HP 배수 1.4→2.4), 옛 저장 최고 섹터·강화 보존.
- 참고: 난이도 곡선이 구 방식(섹터당 +6)보다 크게 완만(섹터당 +1.25)해졌다. 지시서 §4.4 지침대로 기존 수식은 유지했으며, 실제 플레이에서 후반이 물렁하면 계수만 별도 커밋으로 조정 가능.

## Phase C — 항로 보상 계약 (`0bf247f`)

- `logic.baseNodeCoins = 40 + 10×섹터 + 5×열`, `nodeCoinReward = round(base × 타입 배수)`(전투 1.0·보급 0.5·위험 1.2·정예 1.8·수리 0·보스 2.5).
- `logic.nodeModuleGrant`: 전투·위험 3택, 정예 4택(희귀 1장 보장), 보급·수리·보스 없음. `modules.draftOptions`에 희귀 보장(후보 풀에서 1장 먼저 확정).
- 보급 노드 payout ×1.4(단일 round로 경제·모듈·교리·보급 배수 각 1회). 수리 노드 2지선다 UI(긴급 수리 vs 25×섹터 유료 정비, 코인 부족 시 비활성). 지도 노드 정보 라벨.
- 테스트: node-rewards(10). 브라우저 §5.8: 보급 코인 28/35(공식 일치)·모듈 0·payout ×1.4, 위험 3택, 수리 UI + 유료 정비 3택, 지도 라벨 표시. 정예 4택+희귀는 단위 테스트로 검증.

## Phase D — 캠페인 엔딩과 무한 원정 (`2376ca9`)

- `logic.campaignBossId`: 캠페인 = `[B8, B9, B10, B11, B22, B7]`(섹터 6 = B7 하이브 퀸), 엔드리스 = 섹터 7부터 `[B12~B15, B22]` 순환. `sprites.bossDefById` + `bosses.makeBoss/Boss` 보스 ID override로 정체성을 섹터가 아닌 캠페인 순서로 확정.
- 섹터 6 최종 보스 격파 → `winCampaign()`: 정산 + `campaignCleared`/`endlessUnlocked` 저장 + `ui.showVictory`("은하 해방" + 타이틀로/무한 원정 시작/캠페인 다시 시작). 자동 섹터 7 진행 없음.
- 무한 원정: `run.mode`(campaign|endless), 타이틀·승리 화면 버튼(해금 시만), `endlessBest`는 캠페인 기록과 별도 저장. 엔드리스 사망은 기존 실패 보상·격납고 경제 사용.
- `save.js` 필드 병합: `campaignCleared`/`endlessUnlocked`/`endlessBest` 기본값 추가(구버전 저장 무손실).
- 테스트: campaign(7). 브라우저 §6.7(임시 디버그 진입 사용 후 제거): 섹터 6 = B7 하이브 퀸(아트 B7), 격파 → "은하 해방" 승리 화면 + 3버튼, 세이브 플래그 저장, 새로고침 후 무한 원정 버튼 유지, 엔드리스 시작 = 섹터 7 보스 B12 프리즘 타이런트 순환.
