# NEON FLEET Gate 1 재작업 결과 — Codex 반려 결함 대응

작성: Claude(총비서) · 2026-07-19
브랜치: `claude/neon-fleet-overhaul-phase-c-core-loop`
검증 반려서: `docs/2026-07-18-Codex-검증결과-Gate0-Gate1.md`

> Codex가 Gate 1을 반려(차단 6 + 높음 3)했다. 지적의 핵심 — "8분 모드가 사람이 플레이하는 게임이
> 아니라 수치를 재는 자동 재생이었고, 일부 기능은 게임에 연결되지 않은 채 '작동한다'고 보고됐다" —
> 를 전부 수용하고, **플레이 모드와 측정 모드를 분리해 실제 사람이 플레이하는 수직 슬라이스로 다시
> 지었다.** master 미병합·미배포는 유지한다.

## 2. 재작업 후 측정 (측정 모드 `?coreLoopMeasure=1`, 각 4회 평균)

| 빌드 | 공명 기여도 | B22 TTK | 결과 |
|---|---:|---:|---|
| 발칸+레이저 / 레일 스톰 (assault) | **9.4%** | **55초** | clear |
| 발칸+유도 / 마이크로 미사일 포화 (carrier) | **10.4%** | **49초** | clear |
| 레이저+유도 / 시커 빔 (phase) | **9.8%** | **53초** | clear |
| 타이탄+순양함12+드론400 스트레스 | — | — | **내구도 0 사망** |

- 공명 기여도 8~30%: **3빌드 전부 통과** (실제 적용 피해 기준).
- B22 TTK 45~60: **3빌드 전부 통과**.
- 무적 방지: 스트레스 빌드가 긴급 재건 1회를 쓰고도 내구도 0 사망 → **최강 함대도 무적 아님**.

## 3. 사람 플레이 완주 (`?coreLoopTest=1`, 헤드리스 회피 시뮬)

- 시작: H0·내구도 100·무기 1개(시작 선택)·소편대 8.
- 8분 완주: 시작무기·행동변화 10회·두 번째 무기(75s)·지휘 프레임(330s) **모두 사람이 선택**.
- 공명 완성 ~271초(실제 플레이의 발칸 명중/레이저 표식으로), 카펠 프레임 일제사격 실발동(254프레임), **B22 TTK 54초**, 내구도 100→감소(실피해)·긴급 재건 사용.
- 결과 화면: 실제 시작(H0·내구도100)→최종, 무기 2·공명, 피해 비율, 내구도, 다음 설계도.

## 4. 런타임 통합 검증

`window.__nfCoreLoop.integrationCheck()`(브라우저) — 소스 문자열이 아니라 **실제 게임 루프**를 돌려
두 무기 실피해·공명 8~30%·프레임 자동 스킬 실발동·TTK 45~60·내구도 감소·무적 방지·긴급 재건을 단언한다.
(빌드별 개별 실행으로 검증 완료 — assault/carrier/phase 프레임 3종 모두 실발동 확인.)

## 5. 자동 테스트 · 회귀

## 6. 남은 항목 (Gate 2 범위, 정직히 표기)

- **B7 하이브 퀸 보스 랩 TTK(60~90)**: 이번 하네스는 B22 검증 보스만. B7 결전은 Gate 2 캠페인 통합에서 측정(§5.8 지시서도 B7은 25분 캠페인 완성 빌드 기준).
- **B8~B11 대비 측정**: B22가 "섹터 1~4 보스보다 강함"의 정량 비교는 Gate 2 캠페인 통합에서(현재는 적응형 HP+상한으로 TTK 45~60 보장).
- **공개 캠페인 내구도 통합**: 현재 캠페인은 구 드론=체력 모델 유지(§5.2 "Gate 1 통과 전 캠페인 제거 금지"). Gate 2에서 정식 통합.
- 사람 플레이 hull=100 8분 생존 곡선은 실제 사용자 플레이테스트로 최종 조정 필요(현재 헤드리스 회피 기준 생존 가능).
