# 네온 함대 (Neon Fleet) — 게임 설계 문서

- 작성일: 2026-07-05
- 프로젝트 성격: 개인 취미 프로젝트 (회사 업무와 무관)
- 플랫폼: HTML5 웹게임 (모바일 브라우저 + PC 브라우저 겸용)
- 기술: 순수 JavaScript + Canvas 2D, 빌드 도구 없음, GitHub Pages 배포 가능

---

## 1. 목표와 배경

라스트워(Last War: Survival)의 낚시성 광고에 나오는 "게이트 러너 + 부대 증식 + 슈팅" 미니게임을 벤치마킹하여, **낚시 없이 그 게임 자체를 온전히 즐길 수 있는 무료 웹게임**을 만든다.

핵심 근거:
- 게이트 러너 장르는 누적 5억+ 다운로드가 검증된 장르 (Count Masters, Crowd Evolution 등).
- "가짜 광고를 진짜 게임으로" 전략은 이미 성공 사례가 있다 (Hero Wars 기능 편입 → 매출·리텐션 상승, KFC Original Fake Games → 기대 대비 300% 다운로드).
- 원작에 대한 최대 불만은 ① 광고 속 게임이 실제 게임의 일부(약 20%)에 불과, ② 과금 압박. → 본작은 **미니게임이 100%, 과금·광고 0**이라는 것이 정체성.

### 벤치마킹에서 얻은 설계 원칙

| 장르의 성공 요인 | 반영 |
| --- | --- |
| 곱셈 게이트의 극적·즉각적 결과 | +N, xN 게이트 유지, 통과 즉시 시각·수치 피드백 |
| 30~60초의 짧은 판 | 한 판 40~60초, 실패 후 즉시 재시작 |
| 눈에 보이는 무리 성장 | 드론 떼가 커지는 시각 연출 (수백 기는 숫자+무리 표현) |
| 조작이 단순함 (좌우 이동뿐) | 드래그/마우스/방향키 하나로 통일 |

| 장르의 비판 요인 | 반영 |
| --- | --- |
| 광고 폭탄 | 광고 없음 |
| 너무 쉬움 | 리스크-리워드 배치 규칙 + 실패 가능한 난이도 곡선 |
| 반복적, 하루면 소진 | 청크(구간 패턴) 랜덤 조합 + 스테이지 해금 + (추후) 엔드리스 모드 |
| 강화만 하면 무조건 승리 | 강화 수치와 스테이지 난이도를 함께 상승 |

## 2. 게임 컨셉

**"드론 편대를 불려서 모선을 격파하라"** — 세로 스크롤 트랙 위에서 드론 편대가 자동 전진·자동 사격한다. 플레이어는 좌우 이동만으로 크리스탈·게이트·적을 선택하며 편대를 키우고, 트랙 끝의 모선(보스)을 파괴하면 클리어.

### 테마 변환표 (원작 디자인 카피 회피)

| 원작 요소 | 네온 함대 | 표현 방식 |
| --- | --- | --- |
| 병사 부대 | 소형 드론 편대 | 청록 네온 삼각형/원 + 글로우 |
| 좀비 무리 | 외계 크리처 (전진해 내려옴) | 보라/녹색 네온 불규칙 도형 |
| 정지 장애물 | 운석 (제자리 또는 느린 하강) | 회색+주황 균열 발광 |
| 숫자 술통 | 에너지 크리스탈 (숫자 표시) | 육각 크리스탈 + 숫자 |
| 파랑/빨강 게이트 | 워프 게이트 (청록 +/x, 마젠타 -/÷) | 반투명 발광 사각 필드 |
| 무기 아이템 | 화력 모듈 (일정 시간 강화) | 금색 별/캡슐 |
| 빨간 깃발 위험구역 | 전자기 폭풍 존 | 붉은 파동 이펙트 |
| 보스 | 모선 (화면 상단, HP바) | 대형 네온 함선 실루엣 |

모든 그래픽은 도형 + 네온 글로우로 코드에서 직접 그린다(이미지 파일 불필요). 원작의 3D 카툰 스타일과 시각적으로 완전히 다르므로 디자인 카피 문제가 없다.

## 3. 코어 게임플레이 (한 판의 규칙)

### 3.1 조작
- 모바일: 화면 아무 곳이나 터치 후 좌우 드래그 → 편대가 따라옴
- PC: 마우스 좌우 이동 또는 ←/→ 방향키
- 조작은 이것이 전부. 사격은 자동.

### 3.2 편대 (플레이어)
- 시작 드론 수: 3기 (강화로 증가 가능)
- 드론 수만큼 화력이 비례 증가. 개별 드론은 초당 2발, 탄환 데미지 1 (강화로 증가).
- 화면 표시: 최대 60기까지는 개별 드론을 그리고, 그 이상은 무리 크기 고정 + 편대 위에 수 표시 (예: `x347`).
- 편대 폭은 드론 수에 따라 커진다 → 커질수록 나쁜 게이트/적을 피하기 어려워지는 자연스러운 난이도 상승.

### 3.3 트랙 요소
1. **에너지 크리스탈**: 숫자 N 표시. 총알에 맞으면 N이 깎이고, 0이 되면 파괴되며 **원래 숫자만큼 드론 획득**. 파괴 전에 지나치면 소멸(획득 실패). 큰 크리스탈일수록 오래 쏴야 하므로 "일찍 조준을 시작할지" 선택이 생긴다.
2. **워프 게이트**: 좌우 한 쌍으로 등장. 종류: `+N`(고정 증가), `xN`(배수), `-N`(감소), `÷N`(나눗셈). 통과하는 순간 편대 수에 즉시 적용. 편대 폭이 넓으면 양쪽에 걸칠 수 있으며, **편대 중심이 속한 쪽 하나만 적용**된다.
3. **외계 크리처**: HP를 가지고 아래로 전진. 편대와 접촉 시 `크리처의 남은 HP만큼 드론 소멸` 후 크리처도 소멸.
4. **운석**: 제자리 또는 느린 하강. 접촉 시 동일 규칙. 파괴 시 소량 코인 드랍.
5. **화력 모듈**: 획득 시 10초간 탄환 데미지 2배.
6. **전자기 폭풍 존**: 트랙 일부 폭을 차지하는 붉은 존. 지나가는 동안 초당 드론 -10% 소실. 큰 보상(대형 크리스탈)이 존 안쪽에 배치되는 경우가 있다.
7. **모선 (보스)**: 트랙 끝. HP바 표시. 편대가 사거리에 들어오면 자동 교전. 모선은 주기적으로 크리처 소환 + 느린 탄을 발사(탄에 맞으면 드론 -5). HP 0 → 클리어.

### 3.4 승패
- **클리어**: 모선 파괴. 결과 화면: 획득 코인, 남은 드론 수, 최고 기록 갱신 표시, 재미용 가상 퍼센타일 문구("전체 파일럿 상위 12%!").
- **실패**: 드론 0기. 실패 화면은 0.5초 안에 나타나고 [다시 도전] 버튼 1탭으로 즉시 재시작.

### 3.5 리스크-리워드 배치 규칙 (난이도의 핵심)
- 안전한 경로에는 작은 보상(+5 게이트, 소형 크리스탈).
- 큰 보상(x2 게이트, 대형 크리스탈)은 위험 요소(크리처 무리, 폭풍 존) 근처에 배치.
- 나쁜 게이트(-N, ÷N)는 좋은 게이트와 쌍으로 등장시켜 "반드시 선택"하게 만든다.

### 3.6 청크(구간 패턴) 시스템
- 트랙은 고정 길이 청크를 이어붙여 생성한다.
- 청크는 손으로 설계한 패턴(요소 배치 청사진)이며 난이도 태그(easy / mid / hard)를 가진다.
- 판 진행도에 따라 추첨 풀이 easy → mid → hard로 이동: 판 초반은 성장 위주, 후반은 위협 위주.
- MVP 목표: 청크 15개 이상 (easy 5, mid 6, hard 4).

### 3.7 수치 초안 (구현하며 조정)

| 항목 | 값 |
| --- | --- |
| 논리 해상도 | 폭 480 고정, 높이는 화면 비율에 맞춤 (세로형) |
| 판 길이 | 청크 10개 + 보스 구간 ≈ 45~60초 |
| 스크롤 속도 | 초당 220px (판 후반 +10%) |
| 시작 드론 | 3기 |
| 드론 발사 속도 | 초당 2발, 데미지 1 |
| 크리스탈 크기 | 소 10~20, 중 40~80, 대 150~400 |
| 게이트 값 | +5~+30, x2~x3, -10~-50, ÷2 |
| 크리처 HP | 소형 5, 중형 20, 대형 60 |
| 모선 HP | 스테이지 1 기준 800 |

## 4. 화면 흐름

```
[타이틀] → [플레이] → 클리어 → [결과: 코인/기록/한 판 더]
              ↓ 실패            ↓
          [실패: 다시 도전] ← ← ┘ (2차: 결과 → 격납고(상점) → 스테이지 선택)
```

MVP의 화면은 3개: 타이틀, 플레이, 결과(성공/실패 겸용). 모두 하나의 Canvas 위에 상태 전환으로 구현한다.

## 5. 메타 시스템 (2차 범위)

### 5.1 코인과 영구 강화 (격납고)
- 코인 획득: 클리어 보상 + 판 중 운석 파괴 드랍. 실패해도 진행도 비례 일부 지급 (실패가 완전 헛수고가 되지 않게).
- 강화 4종 (각 레벨 10단계, 비용은 단계마다 상승):
  1. 시작 드론 수 (3 → 최대 13)
  2. 탄환 데미지 (1 → 최대 3)
  3. 발사 속도 (초당 2 → 최대 4)
  4. 코인 획득량 (+10%/단계)
- 밸런스 원칙: 스테이지 N의 권장 전투력을 정해두고, 강화를 적당히 하면 도전적, 만렙이어도 조작 실수는 벌 받게 유지.

### 5.2 스테이지 해금
- 스테이지 1~10. 클리어 시 다음 스테이지 해금.
- 스테이지가 오를수록: 모선 HP 증가, hard 청크 비중 증가, 크리처 HP 상승, 새 요소 등장(폭풍 존은 3부터, ÷ 게이트는 4부터 등).

### 5.3 저장
- localStorage에 JSON 저장: 코인, 강화 레벨, 해금 스테이지, 스테이지별 최고 기록.
- localStorage 사용 불가 환경(시크릿 모드 등)에서는 게임은 정상 동작하되 "저장이 꺼져 있음" 안내만 표시.

## 6. 기술 아키텍처

빌드 도구 없이 ES 모듈로 구성. `index.html`을 열면 바로 실행된다.

```
neon-fleet/
├── index.html          # Canvas + 모듈 로드
├── css/style.css       # 전체화면/세로 레이아웃, UI 오버레이
├── js/
│   ├── main.js         # 진입점, 게임 루프(requestAnimationFrame), 상태 머신(타이틀/플레이/결과)
│   ├── balance.js      # 모든 밸런스 수치를 한 곳에 (조정 편의)
│   ├── input.js        # 터치/마우스/키보드 → "목표 x좌표" 하나로 통일해 전달
│   ├── entities.js     # 편대, 탄환, 크리스탈, 게이트, 크리처, 운석, 모듈, 모선 (각자 update/draw)
│   ├── chunks.js       # 청크 패턴 정의 + 난이도별 추첨 + 트랙 생성
│   ├── collision.js    # 충돌 판정 (원-원, 원-사각)
│   ├── render.js       # 배경(스타필드), 네온 글로우 헬퍼, HUD(드론 수/보스 HP/진행 바)
│   ├── ui.js           # 타이틀/결과/실패 오버레이 (DOM 버튼)
│   └── save.js         # localStorage 래퍼 (실패 안전)
```

- 각 모듈은 한 가지 책임만 가지며, `balance.js`의 수치만 바꿔 밸런싱할 수 있게 한다.
- 성능 목표: 중급 안드로이드 폰 브라우저에서 60fps. 화면 밖 개체 즉시 제거, 탄환 수 상한(동시 400발) 초과 시 오래된 탄 제거, 드론 개별 렌더 60기 상한.
- 좌표계: 논리 폭 480 고정, 실제 화면 크기에 맞춰 배율. PC 와이드 화면에서는 중앙 세로 기둥 형태로 표시.

### 에러 처리
- localStorage 접근은 try-catch로 감싸고 실패 시 메모리 저장으로 대체.
- 게임 루프에서 프레임 간격이 비정상(백그라운드 탭 복귀 등)이면 dt를 최대 50ms로 잘라 물리 폭주 방지.

## 7. 개발 범위

### MVP (1차 완성 목표)
- [ ] 스테이지 1 한 판 전체: 이동, 자동 사격, 크리스탈, 게이트(+/x/-), 크리처, 운석, 화력 모듈, 모선전
- [ ] 청크 15개 + 난이도별 추첨
- [ ] 타이틀/결과/실패 화면, 즉시 재시작
- [ ] 모바일 터치 + PC 마우스/키보드
- [ ] 최고 기록 localStorage 저장
- [ ] 중급 폰 브라우저 60fps

### 2차 (재미 검증 후)
- 격납고(강화 상점) + 코인 경제
- 스테이지 2~10 해금, 신규 요소(폭풍 존, ÷ 게이트, 신규 크리처)
- 효과음/배경음 (WebAudio 생성음)
- 엔드리스 모드 + 가상 퍼센타일
- GitHub Pages 배포

### 하지 않는 것 (YAGNI)
- 서버/실제 랭킹, 계정, 실제 결제·광고, 3D 그래픽, 앱스토어 배포 (원하면 추후 웹뷰 포장)

## 8. 테스트 및 검증

- 핵심 로직(게이트 연산, 크리스탈 카운트다운, 충돌, 청크 추첨, 저장)은 브라우저 없이 실행 가능한 순수 함수로 작성하고 Node 기반 단위 테스트를 붙인다.
- 렌더링/조작은 수동 검증 체크리스트로 확인: PC 크롬, 안드로이드 크롬에서 ① 60fps ② 드래그 추적 정확성 ③ 게이트 적용 값 ④ 실패→재시작 1탭 ⑤ 새로고침 후 기록 유지.
- 밸런스 검증: "강화 없이 스테이지 1을 3~5판 안에 클리어 가능"을 기준으로 청크/수치 조정.
