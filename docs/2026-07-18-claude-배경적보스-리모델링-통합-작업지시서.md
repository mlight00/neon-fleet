## 1. 플레이테스트 피드백과 이번 목표

사용자가 섹터 6까지 직접 완주한 뒤 남긴 판단을 단일 기준으로 삼는다.

### 만족한 부분 — 유지

- 기함 티어별 이미지 변화
- 무기 체계와 강화 단계별 이미지 변화

위 두 영역은 이번 작업에서 다시 디자인하지 않는다. 크기, 색, 파일, 합성 규칙도 특별한 결함이 없는 한 유지한다.

### 즉시 수정할 문제

1. 배경이 위쪽으로 흘러 함대가 전진하는 것이 아니라 뒤로 가는 듯하다.
2. 섹터 5 보스 네온 아비터가 페이즈 진행 중 여러 이미지 조각으로 분해되어 보인다.
3. 섹터별 배경이 사실상 같은 이미지의 색상 변형이라 구역을 돌파한다는 체감이 약하다.
4. 기존 일반 적과 섹터 1~4 보스가 새 기함·무기 아트의 완성도와 톤에 맞지 않는다.

### 완료 후 플레이어가 느껴야 하는 변화

- 배경의 모든 큰 구조물이 화면 위에서 들어와 아래로 빠져 함대가 위로 전진하는 느낌이 난다.
- 섹터가 바뀌면 1초 안에 배경 구조와 색으로 새로운 지역임을 알 수 있다.
- 일반 적은 40~52px에서도 역할이 실루엣으로 구분된다.
- 보스 6종은 이름을 보지 않아도 서로 다른 존재로 읽힌다.
- 섹터 5 아비터는 어떤 페이즈에서도 하나의 거대한 연결형 함선으로 보인다.

### 4.2 현재 섹터 인덱스 결함 수정

현재 `zoneForSector()`는 `(sector - 1) / 2`를 사용해 1~2, 3~4, 5~6을 같은 구역으로 묶는다. `createZoneBackdrop()`의 `zi`도 같은 방식이다. 여섯 구역을 실제로 사용하려면 다음 규칙으로 바꾼다.

```js
const index = Math.min(5, Math.max(1, sector) - 1);
```

- 섹터 1~6은 S1~S6에 1:1 대응한다.
- 섹터 7 이상은 별도 아트가 생기기 전까지 S6에 고정한다.
- 구역 이름, 색, 모티프도 같은 인덱스를 사용한다.

### 4.3 배경이 거꾸로 흐르는 원인과 수정 계약

현재 `drawVerticalArt()`는 다음처럼 오프셋을 뺀다.

```js
const y = i * tileH - offset;
```

`scroll`이 증가할수록 그림이 위로 이동한다. 반면 먼지·파편·항적은 `+ scroll * speed`를 사용해 아래로 이동한다. 한 화면에서 이미지와 입자가 반대로 움직이는 것이 후진 느낌의 직접 원인이다.

수정 후 계약:

- 플레이어 함대는 위를 향한다.
- 월드의 배경 특징점은 화면 위에서 들어와 아래로 빠진다.
- `scroll`이 증가할 때 모든 레이어의 화면 y가 증가한다.
- 속도는 FAR < MID < NEAR 순서를 유지한다.
- 호출부에서 음수 scroll을 넘겨 임시로 맞추지 말고, 방향 정의를 `drawVerticalArt()` 한 곳에 고정한다.

권장 타일 좌표:

```js
const y = i * tileH + offset - tileH;
```

기존처럼 홀수 타일마다 `scale(1, -1)`로 상하 반전하면 위성·왕관·용광로가 거꾸로 반복된다. 신규 배경에는 상하 미러 반복을 사용하지 않는다. 같은 방향의 두 장을 겹치고 경계 8~12%에서 알파 교차하거나, 상·하단의 어두운 부분끼리 소프트 마스크로 잇는다.

## 7. 섹터 5 B22 — 쪼개져 보이는 연출 수정

### 7.1 현재 원인

현재 `NeonArbiter.draw()`는 페이즈에 따라 구조 레이어 전체를 숨긴다.

- 페이즈 1: 고리 + 양팔
- 페이즈 2: 왼팔 전체 소거
- 페이즈 3: 고리와 양팔 전체 소거

큰 부품이 통째로 사라져서 피해를 입은 것이 아니라 이미지 조각이 빠진 모델처럼 보인다.

### 7.2 신규 렌더 계약

신규 `B22.webp`를 모든 페이즈에서 항상 100% 베이스로 그린다. 새 그림의 외곽 고리는 4개 굵은 지지대로 본체에 연결되고, 좌우 무장도 넓은 장갑 뿌리로 선체에 붙어 있다.

- 페이즈 1: 신규 베이스 + 정상 코어 광
- 페이즈 2: 신규 베이스 유지 + 왼쪽 무장 주변 국소 균열·불꽃·채도 저하
- 페이즈 3: 신규 베이스 유지 + 고리 일부 암전·중앙 코어 펄스·균열 증가
- BREAK: 베이스를 없애지 않고 80~120ms 국소 흔들림, 백색 플래시, 파편 VFX

금지:

- `B22_RING`, `B22_ARM_LEFT`, `B22_ARM_RIGHT`를 페이즈에 따라 통째로 숨겨 실루엣을 바꾸는 방식
- 신규 B22 베이스 위에 좌표가 다른 구형 B22 부품을 합성하는 방식
- 페이즈 3에서 중앙 코어만 남기는 방식

기존 부품 파일은 보존하되 신규 B22 렌더에는 사용하지 않는다. 충돌과 STAGGER/BREAK 판정은 그대로 유지한다.

## 8. 섹터 6 B7 — 신규 여왕과 구형 부품 혼합 금지

신규 `B7.webp`는 기존 B7 부품 레이어와 좌표·실루엣이 다르다. `B7_BODY`, `B7_EGG_LEFT`, `B7_EGG_RIGHT`, `B7_CROWN`, `B7_HEART`를 신규 베이스 위에 겹치지 않는다.

- 모든 전투 페이즈에서 신규 B7 전체 베이스를 유지한다.
- 좌우 산란낭 상태는 해당 영역의 국소 발광, 균열, 암전, 연기 VFX로 나타낸다.
- 왕관 파괴는 왕관 영역의 금빛 팁 암전과 균열로 나타내며 몸체 전체를 숨기지 않는다.
- 심장 노출은 중앙 심장 주변 장갑 펄스와 밝기 증가로 표현한다.
- `B7_ESCAPE`는 최종 분리 순간에만 별도 개체로 사용할 수 있다.

## 9. 로딩과 성능

- 배경은 현재 섹터 한 장만 필수 로드하고 다음 섹터를 보스전 또는 통과 연출 중 프리로드한다.
- 현재 캠페인 보스만 보스 경고 시작 때 프리로드한다.
- 일반 적은 현재 로스터 슬롯만 우선 로드하되, 전체 12개 동시 로드도 용량 검토 후 허용한다.
- 없는 파일 요청과 구형/신형 자산 이중 요청을 만들지 않는다.
- 신규 런타임 24개 총용량은 약 7MB이며, 정확한 값은 매니페스트를 사용한다.
