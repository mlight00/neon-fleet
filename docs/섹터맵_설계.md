# 섹터 분기 맵 — 설계 기록

## 목표 (사용자 승인 A안)
연속 스테이지(매판 보스) → **섹터 분기 맵**(슬레이 더 스파이어식). 섹터당 갈림길 노드맵을 골라 진행 →
섹터 보스 → 다음 섹터. 보스 빈도 적정 · 중간 정비 · 선택의 자유 · 리플레이성.

## 핵심 통찰 (저위험 통합)
`run.stage`를 **난이도 카운터로 유지**한다: `stage = (sector-1)×(노드수) + node.col`. 기존 적HP·보스·멀티보스·
크기·로스터·게이트 스케일이 전부 `r.stage`에 걸려 있으므로 **전투 시스템은 그대로 재활용**. 맵은 그 위의
**선택·라우팅 레이어**만 담당.

## 데이터 모델
- `run.sector`(1+), `run.map`(노드 그래프), `run.node`(현재 노드).
- 맵 = 열(column) 배열. col 0=진입(자동), col 1..D-1=선택 노드 2~3개, col D=보스 1개. D≈5(≈"5스테이지마다 보스").
- 노드 `{ id, col, row, type, next:[다음열 row들] }`. 분기: 각 노드→다음열 1~2개 연결(도달성·역연결 보장).
- 생성=`generateSectorMap(sector, rng)` 순수함수(logic.js, 테스트). 시드 재현 가능.

## 노드 종류 (기존 청크/시스템 재활용, 새 아트 불필요)
| 아이콘 | type | 인카운터 |
|---|---|---|
| ⚔️ | combat | 일반 적 트랙(기본) |
| ☠️ | elite | 강화 트랙(변이 강제+미니보스)·보상 큼 |
| ☄️ | hazard | 장애물(debris/mine) 트랙, 적 없음 |
| 💎 | supply | 크리스탈·캡슐 안전 트랙 |
| 🔧 | repair | 정비: 드론 회복 + 모듈 드래프트(무료 pit stop) |
| 👑 | boss | 섹터 보스 |
(❓event=위험보상 선택은 Phase 2)

노드→청크 필터(내용 기준, 신규 청크 불필요):
- supply=`isSafeChunk && (crystal|capsule)`, hazard=`(debris|mine) && !creature`, combat=위협 포함, elite=combat+affix 부스트.

## 흐름 (상태기계 변경)
- `state='map'` 신설(게임 정지, 맵 오버레이 표시).
- startPlay → newExpedition(런 초기화, **트랙 안 만듦**) → `generateSectorMap(1)` → state 'map'.
- 노드 클릭 → `enterNode(node)`: `r.stage` 설정 + `buildEncounter(node)`(타입별 트랙) → state 'play'.
  - repair 노드=트랙 없이 회복+드래프트 → 맵 복귀. boss 노드=트랙 끝에 보스 스폰.
- 인카운터 클리어(기존 flythrough 끝) → 노드 done → **맵 복귀**. 보스 노드 클리어 → 섹터 완료 → 다음 섹터(새 맵, 더 어려움).
- 기존 `advanceStage`(자동 다음스테이지) → `onEncounterClear`(맵 복귀/섹터전환)로 대체.

## buildEncounter(node) (buildStage 일반화)
- `r.stage = (sector-1)*NODES + node.col`; `r.isBossNode = node.type==='boss'`.
- 트랙 길이: combat/elite ~7청크, hazard/supply ~4, boss ~4+보스. 타입 필터 적용.
- 보스 스폰 게이트: 트랙 끝 보스는 `r.isBossNode`일 때만(기존 `traveled>=totalTrack` 지점).
- 비전투(repair)=트랙 스킵, 즉시 회복+드래프트.

## 맵 UI (ui.showSectorMap)
- DOM 오버레이. 열별로 노드 아이콘 배치 + 연결선. **현재 노드에서 도달 가능한 다음 노드만 클릭 가능**(하이라이트).
- 지나온 노드=흐리게, 보스=상단. 클릭 → onPick(node).

## 구현 순서 (단계별 검증)
1. logic.generateSectorMap + 순수함수 테스트(열구조·분기·도달성·보스 1개).
2. ui.showSectorMap(맵 데이터, onPick).
3. main.js 흐름 리팩터(newExpedition/enterNode/onEncounterClear/buildEncounter + 'map' state + 보스 게이트).
4. 헤드리스: 맵생성→노드진입→클리어→맵복귀→보스→다음섹터. 스모크 무충돌.
5. 커밋·배포.

## 튜닝/주의
- 노드수 D·타입 분포·트랙 길이·repair 회복량은 실플레이 튜닝.
- 기존 continuous 대비 큰 변경 — 사망/재시작·격납고(사망시)·최고기록 저장은 sector 기준으로 유지.
- Phase 2: event 노드, 크레딧 상점형 격납고, 맵 비주얼 강화, 섹터별 배경/보스 테마.
