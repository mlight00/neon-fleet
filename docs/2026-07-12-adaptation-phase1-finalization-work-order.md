## 0. 작업 원칙

1. 이번 작업은 **Phase 1 마감 작업**이다. 신규 무기·교리·적·보스·모듈을 추가하지 않는다.
2. 기존 저장 형식과 진행 기록을 변경하지 않는다.
3. `bankStack`, 저장 마이그레이션, hazard 필터, 강등 안전망을 건드리지 않는다.
4. 패러사이트에 완전 면역을 만들지 않는다. 일반 공격으로도 충분히 처치 가능해야 한다.
5. `entities.js`와 `bosses.js`를 추가로 분할하거나 구조를 재편하지 않는다.
6. 신규 로직은 가능하면 `adaptive-logic.js`의 순수 함수로 만들고 자동 테스트로 보호한다.
7. 테스트를 실제 구현보다 느슨하게 맞추지 않는다. README·결과서·코드가 같은 동작을 설명해야 한다.
8. 각 커밋은 동작 수정, 테스트, 문서 정리 단위로 분리한다.

## 2. P2 — 게이트 패러사이트 방어와 랜스 강습 관통 구현

### 2.1 현재 문제

README에는 아래 효과가 적혀 있다.

```text
랜스 강습 3단 이상 차지 랜스가 프리즘 방어막·패러사이트 방어 무시
```

프리즘 워든은 `lanceCtx.pierceDefense`를 사용하지만 `GateParasite.hitByBullet()`은 세 번째 문맥 인자를 받지 않으며 방어 효과도 없다. 따라서 현재 패러사이트는 일반탄, 일반 랜스, 랜스 강습 3단 랜스를 구분하지 않는다.

### 2.2 대상 파일

- `js/balance.js`
- `js/adaptive-logic.js`
- `js/adaptive-enemies.js`
- `js/entities.js`는 랜스 문맥 계약 확인만 한다. 불필요한 추가 수정은 금지한다.
- `tests/adaptive-logic.test.mjs`
- 신규 `tests/adaptive-enemies.test.mjs`

### 2.3 구현 사양

`BAL.adaptiveEnemies.gateParasite`에 일반 피해 감소율을 추가한다.

권장 초기값:

```js
armorReduce: 0.35
```

피해 규칙:

```text
일반 탄환                  → 원래 피해의 65%
도탄·분열·폭발             → 원래 피해의 65%
일반 교리의 차지 랜스       → 원래 피해의 65%
랜스 강습 1·2단            → 원래 피해의 65%
랜스 강습 3단 이상          → 원래 피해의 100%
```

`adaptive-logic.js`에 DOM·world와 무관한 순수 함수를 추가한다.

권장 인터페이스:

```js
export function parasiteDamageMult(ctx, armorReduce) {
  const pierces = !!(ctx?.lance && ctx?.pierceDefense);
  return pierces ? 1 : 1 - armorReduce;
}
```

`GateParasite.hitByBullet()`은 다음 계약을 따른다.

```js
hitByBullet(dmg, world, ctx = null)
```

- `parasiteDamageMult()` 결과를 피해에 한 번만 곱한다.
- 피해 감소 상한은 기존 원칙대로 70%를 넘지 않는다.
- `this.dead`인 경우 즉시 반환해 보상·코인이 중복 지급되지 않게 한다.
- 관통 시 기존 네온 색과 구분되는 짧은 흰색 파열 효과를 표시한다.
- 일반 공격으로도 처치 가능해야 하며 무기 완전 면역은 금지한다.
- 감염 게이트 반전 규칙과 정화 보상 10드론은 변경하지 않는다.

### 2.4 플레이 의도

패러사이트의 대응법은 최소 세 가지가 된다.

1. 일반 공격으로 일찍 집중 사격해 정화한다.
2. 감염되지 않은 반대 레인을 선택한다.
3. 랜스 강습 3단 이상으로 방어를 관통해 빠르게 제거한다.

### 2.5 필수 테스트

1. `ctx === null`이면 피해 배수가 `0.65`다.
2. 일반 탄환 문맥이면 피해 배수가 `0.65`다.
3. 일반 랜스와 랜스 강습 1·2단에 해당하는 `pierceDefense:false`는 `0.65`다.
4. 랜스 강습 3단 이상 문맥인 `pierceDefense:true`는 `1`이다.
5. 방어 감소율이 100% 면역을 만들지 않는다.
6. 동일 기본 피해에서 관통 랜스가 일반 공격보다 더 큰 실제 HP 감소를 만든다.

## 3. P2 — 스캐빈저 보상·예약·도주 회귀 테스트

### 3.1 현재 상태

`Crystal`과 `DronePod`의 `getDroneReward(world)` 및 스캐빈저의 실수령 보상 저장은 현재 코드에서 정상 동작한다. 실측 기준:

```text
크리스탈 value 100
정상 지급: round(100 × 0.32) = 32
스캐빈저 회수: round(32 × 1.5) = 48
```

그러나 이 핵심 P1 경로를 직접 검증하는 자동 테스트가 없다.

### 3.2 대상 파일

- `js/adaptive-enemies.js`
- `js/entities.js`
- 신규 `tests/adaptive-enemies.test.mjs`

### 3.3 구현 지침

- 현재 `getDroneReward(world)` 공통 계산을 유지한다.
- 스캐빈저가 `reward` 원시값으로 되돌아가지 않게 한다.
- `Scavenger.hitByBullet()` 시작부에 이미 죽은 개체의 중복 처리를 차단한다.

권장 형태:

```js
hitByBullet(dmg, world) {
  if (this.dead) return;
  // 기존 피해 및 보상 처리
}
```

- 보상 지급 직후 `stored`를 0으로 비우거나 지급 완료 플래그를 남겨 방어적으로 한 번만 지급되게 한다.
- 엔진의 `e.dead` 필터에만 의존하지 않는다.

### 3.4 필수 테스트

최소 다음 테스트를 `tests/adaptive-enemies.test.mjs`에 추가한다.

1. `Crystal(100)`의 기본 실수령이 32다.
2. `podRewardMult`와 `rewardGainMult`가 `Crystal`과 `DronePod`에 동일하게 적용된다.
3. 스캐빈저가 `Crystal(100)`을 훔칠 때 원시값 100이 아니라 실수령 32를 저장한다.
4. 해당 스캐빈저 처치 시 48만 지급한다.
5. 사망 처리를 두 번 호출해도 48이 한 번만 지급된다.
6. 훔치지 않은 스캐빈저를 처치하면 드론 보상 없이 기본 코인만 지급한다.
7. 보상을 들고 화면 밖으로 도주하면 드론 보상이 지급되지 않는다.
8. 두 스캐빈저가 같은 보상을 동시에 `claimedBy`로 예약하지 못한다.
9. 예약한 스캐빈저가 보상 강탈 전에 죽으면 예약이 해제된다.

테스트 world는 최소 스텁으로 구성하되 실제 `Crystal`, `DronePod`, `Scavenger` 클래스를 사용한다. 계산식만 복사한 가짜 테스트는 금지한다.

## 4. P2 — 패러사이트와 감염 게이트 통합 회귀 테스트

### 대상 파일

- `js/adaptive-enemies.js`
- `js/entities.js`
- `tests/adaptive-enemies.test.mjs`

### 필수 테스트

1. 패러사이트 생성 시 지정 레인의 `gate.corruptSide`가 설정된다.
2. 패러사이트 생존 상태에서 감염 레인을 통과하면 게이트 연산이 반전된다.
3. 패러사이트 처치 시 `gate.corruptSide`가 `null`로 복구되고 정화 보상 10드론이 한 번 지급된다.
4. 정화 후 게이트 통과 결과가 원래 연산과 일치한다.
5. 반대편 비감염 레인은 원래 연산을 유지한다.
6. 부모 게이트가 `dead` 또는 `applied` 상태가 되면 자식 패러사이트가 다음 업데이트에서 정리된다.
7. 부모 소멸 정리는 정화 보상이나 코인을 지급하지 않는다.

가능하면 실제 `GatePair`와 `GateParasite`를 함께 사용해 부모·자식 통합 동작을 검증한다.

```js
import { Boss as EntityBoss } from '../js/entities.js';
```

## 10. 브라우저 수동 검증

### 화면 크기

- 데스크톱: 1280×720 이상
- 모바일: 390×844

### 필수 시나리오

1. 타이틀 → 출격 → 맵 → 전투 진입
2. 일반 공격으로 패러사이트 처치 가능 확인
3. 일반 차지 랜스와 랜스 강습 3단 관통 피해 비교
4. 패러사이트 생존 시 감염 게이트 반전 확인
5. 정화 후 원래 게이트 연산 복구 확인
6. 스캐빈저가 보상을 훔치고 도주하면 미지급 확인
7. 스캐빈저 처치 시 정상 실수령의 정확히 ×1.5만 지급 확인
8. 보스 노드 진입 후 보스 등장·사격·피격·격파 흐름 확인

### 화면·오류 기준

- 가로·세로 문서 오버플로 없음
- 콘솔 `error`와 `warning` 없음
- 패러사이트 방어와 관통이 색 변화 또는 파열 효과로 구분됨
- 모바일에서 감염 레인, 패러사이트, 차지 버튼을 구분할 수 있음

브라우저 모듈 캐시 영향을 피하도록 새 탭 또는 커밋 쿼리로 검증한다.
