## 0. 작업 원칙

1. 신규 조작 버튼을 추가하지 않는다.
2. 기존 저장 데이터와 진행 기록을 깨뜨리지 않는다.
3. 신규 시스템은 원정 내부 상태이며 localStorage에 저장하지 않는다.
4. 기존 56개 테스트를 모두 유지한다.
5. 퍼센트 수치만 증가시키는 업그레이드보다 공격 형태와 대응법이 달라지는 업그레이드를 우선한다.
6. 특정 무기 완전 면역은 금지한다. 피해 감소는 최대 70%까지만 허용한다.
7. 신규 적은 최소 두 가지 대응법을 제공한다.
8. 신규 이미지가 없어도 Canvas 네온 도형으로 완전히 플레이 가능해야 한다.
9. 한 커밋에 시스템 여러 개를 뒤섞지 않는다.
10. 각 단계에서 테스트와 브라우저 스모크를 통과한 뒤 다음 단계로 진행한다.

## 1. 플레이 경험 목표

원정의 성장 리듬을 다음처럼 만든다.

```text
첫 무기 선택
→ 첫 전투 후 모듈 선택
→ 무기 Lv3 이후 같은 무기 캡슐로 무기 진화
→ 첫 기함 업그레이드 시 함대 교리 선택
→ 신규 적이 무기 진화·교리 활용을 요구
→ 섹터 2 종료 전 빌드 정체성 완성
```

### 완료 후 플레이어가 느껴야 할 차이

- 발칸은 다수전 또는 정밀 집중형으로 갈린다.
- 레이저는 분열 광역 또는 보호막 절단형으로 갈린다.
- 호밍은 소형 미사일 군집 또는 대형 폭발형으로 갈린다.
- 기함은 드론·차지·기동 중 한 전투 교리에 특화된다.
- 신규 적은 단순 체력벽이 아니라 조준 위치와 공격 수단 전환을 요구한다.

## 2. 공통 데이터 구조

### 2.1 무기 진화 상태

`Squad`에 원정용 상태를 추가한다.

```js
this.weaponEvolutions = {
  vulcan: null,
  laser: null,
  homing: null,
};
this.pendingWeaponEvolution = null;
```

진화 ID:

```text
vulcan_storm
vulcan_needle
laser_prism
laser_cutter
homing_wasp
homing_siege
```

### 2.2 기함 교리 상태

```js
this.doctrine = null; // swarm | lance | phase
this.pendingDoctrine = false;
```

교리는 첫 기함 업그레이드(`tier 0 → 1`) 직후 한 번만 선택한다.

### 2.3 밸런스 데이터

모든 신규 수치는 `js/balance.js`에 둔다.

```js
weaponEvolution: {
  // 진화별 수치
},
doctrine: {
  // 교리별 수치
},
adaptiveEnemies: {
  // 신규 적 수치
},
```

행동 분기에서 숫자를 하드코딩하지 않는다.

### 2.4 정의 파일 권장

신규 정의와 순수 계산은 다음처럼 분리한다.

```text
js/weapon-evolutions.js
js/doctrines.js
js/adaptive-enemies.js
```

`entities.js` 전체 분할은 하지 않는다. 신규 적 클래스만 `adaptive-enemies.js`에 두고 기존 `world` 인터페이스를 사용한다.

## 3. 무기 진화 시스템

### 3.1 진화 발생 조건

다음 조건을 모두 만족할 때 진화 선택창을 연다.

1. 현재 무기 레벨이 `BAL.weapons.maxLv`다.
2. 현재 무기와 같은 색 캡슐을 다시 획득했다.
3. 해당 무기가 아직 진화하지 않았다.
4. 보스 파괴·클리어·패배 연출 중이 아니다.

이미 진화한 무기의 같은 색 캡슐을 다시 획득하면 다음 보상으로 전환한다.

```text
드론 +20 또는 코인 +10
```

보상값은 `BAL.weaponEvolution.duplicateReward`에 둔다.

### 3.2 UI 흐름

`ui.showWeaponEvolution({ weapon, options, onPick })`을 추가한다.

- 카드 2장
- 현재 무기 이름과 진화 이름 표시
- 공격 형태 변화 한 줄
- 장점과 단점 각각 한 줄
- 키보드 방향키 + Space 선택 지원
- 선택 중 게임 완전 정지
- 차지·일시정지 버튼 숨김

선택 예:

```text
레이저 진화

[프리즘 어레이]
관통 후 좌우 분열
다수전 특화

[널 커터]
보호막·적탄 절단
집중전 특화
```

### 3.3 진화 6종

#### A. 폭풍 발칸 — `vulcan_storm`

- 확산각 ×1.25
- 탄환 적중 후 가장 가까운 미피격 적에게 1회 도탄
- 도탄 피해: 원탄의 45%
- 도탄 탐색 반경: 120px
- 도탄 탄환은 다시 도탄하지 않음

구현 주의:

- 적중 시 즉시 직접 피해를 주지 말고 짧은 도탄 탄환 또는 명확한 네온 연결 이펙트를 생성한다.
- 동일 적 중복 도탄 방지를 위해 `hitSet` 또는 대상 ID를 사용한다.

#### B. 니들 개틀링 — `vulcan_needle`

- 확산각 ×0.35
- 발사 횟수 ×1.45
- 발칸 전용 치명타 확률 +8%p
- 탄 크기 15% 축소
- 단일 표적 집중형이지만 관통은 추가하지 않음

#### C. 프리즘 어레이 — `laser_prism`

- 첫 적 적중 후 좌·우 각각 가장 가까운 적에게 분열 광선
- 분열 피해: 원탄의 35%
- 분열 탐색 반경: 150px
- 분열 광선은 추가 분열하지 않음
- 보스에게는 분열 추가 피해가 발생하지 않음

#### D. 널 커터 — `laser_cutter`

- 5번째 레이저 탄마다 강화탄 발사
- 강화탄 폭 ×1.7
- 관통 +2
- 경로 반경 18px 안의 적탄 제거
- 강화탄 피해 ×1.25
- 발사 전 짧은 밝기 변화로 강화탄임을 표시

#### E. 와스프 스웜 — `homing_wasp`

- 기존 미사일 1발 대신 소형 미사일 3발
- 3발 총 피해량 = 기존 1발의 115%
- 각 미사일은 서로 다른 표적을 우선 탐색
- 표적 사망 시 즉시 재탐색
- 동시 미사일 상한: 24
- 폭발 반경 없음

#### F. 시즈 토피도 — `homing_siege`

- 발사 주기 ×0.35
- 발당 피해 ×3.2
- 투사체 크기 ×1.5
- 적중 시 반경 70px 폭발
- 폭발 피해: 직접 피해의 35%
- 보스 직접 타격 피해 추가 +15%
- 이동 속도는 기존 호밍의 80%, 회전력은 70%

### 3.4 무기 진화 HUD

현재 무기 이름 옆에 짧은 진화 이름 또는 아이콘을 표시한다.

```text
레이저 · 프리즘
호밍 · 시즈
```

모바일 390px 폭에서 기존 HUD와 겹치면 아이콘만 표시한다.

### 3.5 무기 진화 테스트

최소 다음 테스트를 추가한다.

1. 무기별 진화 옵션이 정확히 2개다.
2. 여섯 진화 ID가 중복되지 않는다.
3. Lv3 미만에서는 진화가 발생하지 않는다.
4. Lv3에서 같은 무기 캡슐을 다시 획득하면 진화 요청이 발생한다.
5. 이미 진화한 무기는 중복 진화하지 않는다.
6. 도탄·분열이 재귀적으로 다시 발생하지 않는다.
7. 널 커터가 정확히 5번째 탄마다 발생한다.
8. 와스프 3발 총 피해와 시즈 DPS가 목표 범위를 벗어나지 않는다.

## 4. 기함 교리 시스템

### 4.1 선택 시점

첫 기함 업그레이드 직후 다음 프레임에 선택창을 연다.

- `tier 0 → 1`일 때만 `pendingDoctrine=true`
- 선택 중 게임 정지
- 선택 전 추가 업그레이드 진행 금지
- 선택 완료 후 보호막·무적 시간은 기존대로 유지

### 4.2 UI

`ui.showDoctrineDraft({ options, onPick })`을 추가한다.

카드 3장:

```text
🐝 군체 지휘
순양함·드론 특화

⚡ 랜스 강습
차지·정면 돌파 특화

◈ 위상 기동
이동·회피 특화
```

### 4.3 교리 3종 Phase 1 효과

#### 군체 지휘 — `swarm`

- 순양함 화력 ×1.25
- 호위 드론 사격 비중 30% → 40%
- 순양함이 존재할 때 드론 획득량 +10%

#### 랜스 강습 — `lance`

- 차지 속도 ×1.20
- 차지 피해 ×1.15
- 3단 이상 랜스가 프리즘 보호막·게이트 패러사이트 방어를 무시

#### 위상 기동 — `phase`

- 실제 피격 반경 -3px, 최소 12px
- 이동 뱅크 절댓값에 따라 무기 피해 최대 +20%
- 정지 또는 느린 이동 중에는 보너스 없음

### 4.4 적용 원칙

- 교리 효과는 `doctrines.js`의 순수 함수로 계산한다.
- 기존 모듈 효과와 곱연산/합연산 순서를 문서화한다.
- 교리 선택 전에는 모든 배수가 1 또는 0인 중립 상태다.
- 강등되어 스카웃이 되어도 선택한 교리는 원정 동안 유지한다.
- 새 원정 시작 시 교리는 초기화한다.

### 4.5 교리 HUD

기함 이름 또는 모듈 줄 옆에 교리 아이콘 하나를 표시한다.

- 군체: `🐝`
- 랜스: `⚡`
- 위상: `◈`

보스전의 짧은 기함 표시에도 포함한다.

```text
드론 34기 · 순양함 2 · 기함 스트라이커 ⚡
```

### 4.6 교리 테스트

1. 첫 기함 업그레이드에서만 교리 요청이 발생한다.
2. 교리는 한 원정에서 한 번만 선택 가능하다.
3. 강등 후에도 교리가 유지된다.
4. 새 원정에서는 교리가 초기화된다.
5. 군체 교리가 supportPower에만 정확히 적용된다.
6. 랜스 교리가 자동사격 피해를 올리지 않는다.
7. 위상 교리의 피격 반경이 최소값 아래로 내려가지 않는다.
8. 모듈과 교리 배수가 중복 적용되거나 누락되지 않는다.

## 5. 신규 적 3종

신규 적은 `js/adaptive-enemies.js`에 구현한다. 기존 `hitByBullet(damage, world)` 호출과 호환하되 필요하면 세 번째 인자로 탄환 정보를 선택적으로 받는다.

```js
hitByBullet(damage, world, bullet = null)
```

기존 적은 세 번째 인자를 무시해도 정상 동작해야 한다.

### 5.1 프리즘 워든 — `prismWarden`

#### 역할

정면 사격만 유지하는 플레이어에게 측면 조준 또는 차지 사용을 요구한다.

#### 수치 초안

```text
최소 등장 내부 stage: 2
HP: 70
반경: 22
정면 피해 감소: 70%
측면 코어: 좌·우 1개
코어당 HP: 22
코어 오프셋: 중심에서 ±18px
코인: 7
```

#### 동작

- 정면 중앙에 방어막 표시
- 탄환 x 좌표가 중심 코어 범위면 본체 피해 30%
- 좌우 코어 위치에 맞으면 해당 코어 우선 피해
- 두 코어 파괴 시 방어막 해제, 본체 정상 피해
- 차지 랜스는 방어막 감소를 무시하되 코어를 자동 파괴하지는 않음
- 코어 파괴 상태를 색과 균열로 명확히 표시

#### 대응법

- 좌우 이동으로 코어 조준
- 차지 랜스로 정면 관통
- 프리즘 레이저 분열로 양측 압박

### 5.2 크리스탈 스캐빈저 — `scavenger`

#### 역할

안전한 생존과 보상 추격 사이에 선택을 만든다.

#### 수치 초안

```text
최소 등장 내부 stage: 2
HP: 35
반경: 17
접근 속도: 150px/s
도주 속도: 260px/s
훔치기 반경: 28px
훔친 보상 반환 배수: 1.5
코인: 5
```

#### 동작

- 화면 안의 살아 있는 `Crystal` 또는 `DronePod` 중 가장 가까운 대상을 선택
- 대상에 도달하면 해당 보상을 보관하고 대상을 제거
- 이후 화면 상단으로 도주
- 도주 전 처치하면 저장한 드론 보상 ×1.5 지급
- 보상을 훔치기 전 처치하면 기본 코인만 지급
- 대상이 없으면 짧게 머물다 도주

#### 구현 요구

- `Crystal`과 `DronePod`에 읽기 전용 보상 조회 인터페이스를 추가한다.
- 스캐빈저가 제거한 보상이 원래 파괴 보상과 중복 지급되지 않게 한다.
- 여러 스캐빈저가 같은 보상을 동시에 예약하지 못하게 `claimedBy`를 사용한다.

### 5.3 게이트 패러사이트 — `gateParasite`

#### 역할

게이트 선택 전에 조준 우선순위를 만든다.

#### 등장 형태

신규 청크 아이템:

```js
{
  type: 'corruptedGate',
  left: { op: '+', value: 30 },
  right: { op: 'x', value: 2 },
  infectedLane: 1,
}
```

#### 수치 초안

```text
최소 등장 내부 stage: 3
패러사이트 HP: 45
반경: 15
게이트보다 36px 위에 배치
정화 보너스: 드론 +10
```

#### 동작

- 한쪽 게이트 위에 패러사이트 부착
- 살아 있는 상태로 통과하면 감염된 게이트 연산 반전
  - `+N → -ceil(N/2)`
  - `x2 → /2`
  - `-N → -N` 유지
  - `/2 → /2` 유지
- 통과 전 처치하면 원래 값 복구 + 드론 10기
- 감염된 게이트는 맥동·균열·경고 아이콘으로 명확히 표시
- 랜스 강습 3단 이상 랜스는 패러사이트에 추가 피해 또는 방어 무시

#### 구현 권장

- 기존 `GatePair`를 복제하지 말고 선택적으로 corruption 상태를 받을 수 있게 확장한다.
- `GateParasite`가 부모 `GatePair` 참조를 가지고 위치를 동기화한다.
- 부모나 자식이 화면 밖으로 나가면 서로 정리한다.

### 5.4 신규 적 청크·도입

`chunks.js`에 최소 다음 청크를 추가한다.

- prism_lane_choice — 프리즘 워든 1기 + 양측 보상
- scavenger_chase — 크리스탈 2개 + 스캐빈저 1기
- corrupted_gate_intro — 감염 게이트 1개, 주변 적 없음
- prism_scavenger_mix — stage 5+
- corrupted_gate_guard — stage 6+

첫 등장 청크는 해당 신규 적 외의 위협을 최소화한다.

### 5.5 신규 적 테스트

1. stage 최소값 이전 청크에서 등장하지 않는다.
2. 프리즘 정면 피해 감소와 코어 피해가 정확하다.
3. 차지 랜스가 프리즘 방어막 감소를 무시한다.
4. 두 코어 파괴 후 본체가 정상 피해를 받는다.
5. 스캐빈저가 하나의 보상만 예약한다.
6. 스캐빈저 처치 시 보상 ×1.5가 한 번만 지급된다.
7. 스캐빈저 도주 시 보상이 지급되지 않는다.
8. 패러사이트 생존/처치에 따라 게이트 결과가 정확히 달라진다.
9. corruptedGate 부모·자식이 화면 밖에서 누수 없이 정리된다.

## 6. 시스템 연결 순서

### 6.1 월드 요청 큐

엔티티가 직접 DOM UI를 호출하지 않는다. `world` 또는 `run`에 요청 상태를 남기고 `main.js`가 선택창을 연다.

권장 예:

```js
world.requests = {
  weaponEvolution: null,
  doctrine: false,
};
```

`main.update()` 마지막에서 요청을 소비한다.

우선순위:

```text
패배/보스 파괴
> 기함 교리 선택
> 무기 진화 선택
> 일반 모듈 드래프트
```

동시에 두 선택창이 열리지 않아야 한다.

### 6.2 게임 정지 상태

기존 `drafting`을 공통 선택 UI 잠금으로 재사용하거나 `choosing` 상태로 일반화한다.

요구사항:

- 선택창 동안 `update()` 정지
- `requestAnimationFrame` draw 유지
- 일시정지·차지 버튼 숨김
- 입력의 `charging=false` 강제
- 선택 후 이전 플레이 상태로 정확히 복귀

### 6.3 충돌 정보 전달

프리즘 코어와 널 커터를 위해 탄환 충돌 시 엔티티에 탄환 문맥을 전달한다.

```js
e.hitByBullet(b.damage, world, b);
```

기존 클래스가 깨지지 않는지 전체 테스트와 브라우저 스모크로 확인한다.

## 7. 밸런스 목표

### 무기 진화

- 동일 무기 Lv3 대비 단일 표적 이론 DPS 변화: 대체로 -10%~+15%
- 특화 상황에서는 최대 +40% 허용
- 한 진화가 모든 상황에서 다른 진화를 압도하면 안 됨
- 호밍의 명중 안정성 프리미엄을 유지한다.

### 기함 교리

- 해당 전문 분야에서 약 +20~25%
- 비전문 분야에는 직접 보너스를 주지 않음
- 교리 하나가 생존과 공격을 동시에 크게 올리지 않음

### 신규 적

- 첫 등장 시 단독 또는 단순 배치
- 명확한 예고 시간 0.7초 이상
- 프리즘 보호막을 정면 자동사격만으로도 5초 이내 처치 가능
- 스캐빈저 추격 실패가 즉사로 연결되지 않음
- 감염 게이트는 통과 1.2초 전까지 정화 가능해야 함
