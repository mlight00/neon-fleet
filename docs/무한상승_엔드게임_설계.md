# 무한 상승(엔드리스) 엔드게임 개선 — 설계 기록

## 배경 / 피드백 (스테이지 10까지 플레이)
1. **난이도 붕괴**: 스테이지 3 이후 기함이 타이탄(최종)까지 진화하면, 내 화력이 과해지고 적이 적어
   **가운데 방치만 해도 클리어**될 만큼 쉬워짐.
2. **동기 상실**: 보스 5종이 변화 없이 반복 → 이후 스테이지 궁금증 없음 + 진화도 끝나 계속할 이유 없음.

## 방향 (사용자 승인)
**무한 상승** — 스테이지·진화·보스가 무한히 강해지고 달라진다. 목표 = 최고 도달 스테이지 기록 경신.
기존 검증 시스템(오버로드·기함 개성·보스 변주)을 재활용해 한 흐름에 해결(A안).

## 근본 원인
- 적 HP는 화력 비례하되 **6배 상한**(`enemyHpPowerCap 6`) → 화력(오버로드로 무한 증가)이 상한을 압도 → 즉사.
- **적 수 고정**(`spawn.enemyMult 2`) → 깊은 판에도 적이 적음 → 방치 클리어.
- 진화가 **타이탄(6단계)에서 종료**, 이후 오버로드는 눈에 안 보이는 파워만.
- 보스 `(stage-1)%5` 순환 + 변주는 HP·탄만↑(모습·패턴 동일).

## 설계 (6부)

### ① 난이도가 계속 따라오게
- `enemyHpPowerCap`을 **스테이지 비례로 상승**: `cap = 6 + 1.2×(stage-1)` → 깊을수록 적이 화력에 맞춰 단단.
- `spawn.enemyMult`을 **스테이지 비례 증가**: `2 + floor((stage-1)/3)`, 상한 4 → 적이 많아져 **움직여야 생존**.
- 스테이지 HP 배수 소폭 상향(`1+0.5g → 1+0.7g`).

### ② 무한 "승천(昇天)" — 진화 종료 제거
- 타이탄 이후 오버로드 = **『타이탄 ★N』 승천 단계**(Squad.ascension 카운터). HUD 별 개수 표시 + "승천 ★N!" 연출.
- 승천 3회마다 **관통 +1**(눈에 보이는 파워 스파이크). 무한 성장.

### ③ 보스 바퀴마다 시각 변주
- 로스터 바퀴(loop)마다 보스 **색 틴트 변경**(loop1 기본 → loop2 붉게 → loop3 보라 …) + 이름 배지(기존).
- 기존 HP·연사·탄수 증가 유지.

### ④ 보스 크기 스테이지 비례 증가
- `Boss.sizeScale = min(1.7, 1 + 0.06×(stage-1))`. `layout()` 상한 `0.22 → 0.22×sizeScale`, 히트박스 `r×sizeScale`.
- 깊은 보스일수록 화면을 채우는 위압감(다중 보스일 땐 fit 계수로 축소).

### ⑤ 다중 보스 (2~3기 동시)
- `r.boss` 단일 → **`r.bosses` 배열**. `r.boss`=선두(연출·HUD 앵커).
- 수: `stage≥8 → 3, stage≥4 → 2, else 1`.
- X 슬롯 배치(`logicalW×(i+1)/(N+1)`), 각 보스 `homeX` 기준 좌우 이동(중앙 대신), sway 폭 `÷N`.
- HP: 각 = rawHp × 1.4 / N(총 1.4배). 크기: `sizeScale ×0.72`(N>1) → 나란히 맞음.
- 페이즈: 전원 사망 시 bossDeath. 업데이트·충돌·드로·페이드 모두 배열 순회. HUD에 최대 3개 HP바 스택.

## 구현 순서 (2커밋)
- **커밋1**: ①난이도 + ②승천 + ③틴트 + ④크기. (단일 보스 유지, refactor 없음 — 저위험)
- **커밋2**: ⑤다중 보스(`r.bosses` 배열 refactor). 신중히 검증.

## 검증
- `node --test`(기존 50) 유지 + 신규 순수함수(승천·보스수·크기·capScale) 테스트.
- 헤드리스: 깊은 스테이지에서 적 수↑·적 HP↑, 승천 별·관통, 보스 크기·수·틴트, 전원 사망 시 클리어.
- 스모크 다수 프레임 무충돌. 배포 후 Pages 빌드 확인.

## 튜닝 주의
- capPerStage 1.2·enemyMultMax 4·sizeScaleMax 1.7·multiTotalMult 1.4·보스수 임계(4/8)는
  실플레이 피드백으로 조정(과하면 슬로그, 약하면 여전히 쉬움).

## 강등 안전망 (2026-07-09, 테스터 피드백 반영)
**문제**: 화력은 두 축 — ①기함 등급 화력(스트라이커 275·고정·큼) + ②드론 편대 수(늘 한 자릿수).
마이너스 게이트는 총화력에 비례해 커지는데 깎는 대상은 ②드론뿐 → 큰 마이너스는 무조건 드론 0 →
즉사. 기함 275가 생존에 전혀 기여 못 하고, 유저가 기대한 "낮은 기체로 강등"이 코드에 없었음(즉사 뿐).

**해결(A안)**: 드론 전멸 순간 `Squad.onDronesDepleted()`가 상위 파워 한 겹을 잃고 편대를 재건해 **생존**.
- 순서: ①승천(overload) 층이 있으면 그것부터 소모(등급 유지) → ②등급 1단계 강등 → ③최하(스카웃)에서만 진짜 사망.
- 재건: 드론 = 시작 편대 수(`stats.startCount`), 실드+진화무적 부여(연쇄 전멸 방지), 강등/재건 VFX·사운드(evolve).
- 효과: 화력 등급 = 여분의 목숨. 스트라이커 전멸 → 인터셉터+드론8(화력 283→93, 큰 setback이나 생존).
  고등급일수록 잃을 게 많아 긴장 유지 + 즉사 좌절감 제거.

**검증**: 순수 노드 테스트 5시나리오(스트라이커/연쇄/스카웃사망/타이탄+승천/양수델타) + 인게임 통합(월드 effects 실호출·크래시 없음). 단위 51 유지.
