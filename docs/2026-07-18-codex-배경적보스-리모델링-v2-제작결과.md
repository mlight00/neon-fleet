- 제작일: 2026-07-18
- 제작: Codex
- 제작 방식: OpenAI 내장 imagegen + 크로마키 투명화·WebP 후처리
- 상태: 이미지 패키지와 QA 완료 / 게임 코드 통합은 Claude 작업 대기

## 1. 결과 요약

플레이테스트에서 확인된 문제를 기준으로 다음 24개 런타임 이미지를 새로 제작했다.

| 구분 | 수량 | 결과 |
|---|---:|---|
| 섹터 1~6 배경 | 6 | 섹터별 고유 랜드마크와 색 체계, 중앙 전투 레인 확보 |
| 일반 적 | 12 | 공격 역할이 큰 실루엣으로 구분됨 |
| 캠페인 보스 | 6 | B8/B9/B10/B11/B22/B7 전면 재디자인 |
| 합계 | 24 | 런타임 WebP 6,919,942바이트, 약 6.60MiB |

기함과 무기 이미지는 사용자가 만족한 영역이므로 변경하지 않았다.

## 2. 배경 리모델링

### 섹터 서사

1. S1 차가운 항적: 깨진 위성과 청색 궤도 폐허
2. S2 프리즘 묘역: 보라 결정 무덤과 청록 굴절
3. S3 용광로 전선: 항성 제련소와 용융 레일
4. S4 부서진 함대: 인간 함대의 백색 장갑 잔해
5. S5 합창의 장막: 흑자색 생체 갑각과 자홍 신경막
6. S6 왕관 핵: 황금 왕관 구조와 먼 왕실 코어

모든 배경은 중앙 44%를 어둡게 유지하도록 설계했다. 런타임 크롭 후 중앙/가장자리 평균 밝기 비율은 0.153~0.574로, 정한 기준 0.65 미만을 모두 통과했다.

런타임 파일은 768×1536이다. 생성 마스터의 비율을 세로로 늘리지 않고 좌우만 중앙 크롭해 휴대전화 전투 화면에서 반복 간격을 늘렸다.

## 3. 일반 적 리모델링

공통 톤은 흑자색 갑각, 자홍색 생체막, 주황색 기능 코어다. 색보다 실루엣이 먼저 역할을 설명하도록 제작했다.

- B1 창형 정찰체
- B2 쌍낫 절단체
- B3 산란낭 중장체
- B4 생체 저격총
- B5 공성 포대
- B6 육방향 직조기
- B16 탄두 격납 폭격기
- B17 쌍전극 전격체
- B18 궤도 외눈
- B19 전면 방패체
- B20 좌우 격납 수송체
- B21 위상 어긋남 암살체

64px 판독 보드에서 12종의 큰 외곽 형태와 주황 약점 위치가 남는 것을 직접 확인했다.

## 4. 캠페인 보스 리모델링

- B8 리퍼 로드: 비대칭 쌍낫
- B9 볼텍스 마우: 중앙이 빈 회전 포식구
- B10 옵시디언 클로: 형태가 다른 좌우 공성 집게
- B11 보이드 세라프: 6개의 칼날 날개와 깨진 후광
- B22 네온 아비터: 연결형 외곽 고리, 중앙 성채, 좌우 이종 무장
- B7 하이브 퀸: 6개 왕관 뿔, 중앙 여왕, 좌우 융합 산란 금고

특히 B22는 고리와 본체를 4개의 굵은 지지대로 연결하고, 좌우 무장도 넓은 장갑 뿌리로 본체에 붙였다. 새 기본 이미지만 보면 더 이상 조립식 부품이 떨어져 있는 형태로 보이지 않는다. 실제 게임에서도 이 장점을 유지하려면 Claude가 페이즈마다 부품 전체를 숨기는 현재 코드를 제거해야 한다.

## 5. 후처리

1. 생성 원본은 순수 초록 배경 보존본으로 저장
2. `remove_chroma_key.py`의 border 자동 샘플링 사용
3. soft matte, despill로 초록색 가장자리 제거
4. 투명 영역 기준 크롭과 4.5% 안전 여백
5. 일반 적 최대 512px, 보스 최대 1024px의 무손실 투명 WebP 제작
6. 배경은 768×1536 WebP quality 88로 제작

자동 검사에서 모든 스프라이트에 유효한 알파 영역이 있고, 완전 투명 배경과 부드러운 가장자리 픽셀이 존재했다. 의심 녹색 픽셀은 각 이미지 0~280개로 극소량이며 QA 보드 육안 검사에서 초록 테두리는 보이지 않았다.
