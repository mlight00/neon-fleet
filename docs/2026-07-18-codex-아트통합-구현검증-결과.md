## 1. 결론

클로드가 질문 문서만 남기고 멈춘 지점에서 작업을 이어받아, 신규 WebP 아트를 실제 게임 코드에 연결했다.

이번 통합에서 확인한 핵심 결과는 다음과 같다.

1. 기함은 H0→H5가 긴 변 34→140px로 단조 증가하며, H5가 H2보다 확실히 크고 위엄 있게 보인다.
2. 발칸·레이저·호밍의 기본형과 분기 6종에 서로 다른 마운트·발사체가 연결됐다.
3. H2는 무기 계열에 따라 Assault/Carrier 프레임이 교차 전환된다.
4. B7 하이브 퀸은 세 마리 복제 보스가 아니라 한 개체의 4단계 보스로 바뀌었다.
5. B22와 B7은 합본 이미지 대신 정렬된 부품 레이어를 단계별로 합성한다.
6. 새 배경, 인트로 5장, 통신 초상, 타이틀 락업, 업그레이드 VFX가 런타임에 연결됐다.
7. 자동 테스트 279건과 WebP 67개 디코드 검증이 모두 통과했다.

## 3. 구현 내용

### 기함과 호위 위계

- `A1~A6`를 신규 H0~H5 WebP로 교체했다.
- 표시 긴 변을 `34 / 50 / 68 / 88 / 112 / 140px`로 고정했다.
- 순양함은 30px, 드론은 18px로 독립 고정해 기함이 커져도 호위가 함께 커지지 않는다.
- 보이는 선체 크기와 실제 피격 핵 반경을 분리해, 함선은 웅장해지지만 회피 난이도가 크기에 비례해 악화되지 않게 했다.
- H2 Assault/Carrier 프레임은 180ms 알파 교차 전환한다.

### 무기

- 기본 3종과 분기 6종, 총 9종의 마운트·발사체를 연결했다.
- 플레이어 일반탄, 도탄, 분열탄, 유도탄이 파생될 때도 해당 아트 계보가 유지된다.
- 이미지 로드 실패 시 기존 Canvas 도형이 폴백으로 남는다.
- 기함 승급과 무기 진화에 신규 점화·장갑 결합·무기 진화·티어 상승 VFX를 사용한다.

### 적과 보스

- B16~B21을 신규 역할 실루엣으로 교체했다.
- B22는 chassis/ring/양팔/core/crack을 단계별 합성하고 BREAK VFX를 표시한다.
- B7은 다음 네 단계로 재구성했다.
  1. 좌우 산란낭과 소환 패턴
  2. 산란낭 파괴 후 왕관 방어와 안전 통로 패턴
  3. 왕관 파괴 후 심장 노출과 추적 3연사
  4. 탈출 코어 분리와 빠른 최종 추격
- B7과 B22가 후반 섹터에서도 1기만 등장하도록 순수 로직 함수와 자동 테스트를 추가했다.

### 배경·스토리·브랜딩

- 섹터 1~4는 Helios, 섹터 5~6은 Hive FAR/MID 배경을 상하 미러 반복한다.
- 인트로 5장과 ECHO-7/하이브 퀸 초상을 실제 이미지로 교체했다.
- 중복 CSS 우주선 그림을 제거하고 스토리 하단 텍스트 영역을 유지했다.
- 타이틀의 중복 텍스트를 제거하고 승인된 `title_lockup.webp`를 사용한다.
- 앱 아이콘도 신규 WebP로 교체했다.

## 7. 남은 확인 사항

코드·이미지 통합 자체는 완료했다. 다만 다음은 수치가 아니라 사람이 직접 느껴야 하는 항목이므로 실제 장시간 플레이가 필요하다.

1. H2→H5 승급 순간의 체감이 충분히 강한지
2. 무기 분기 후 발사음과 화면 흔들림까지 포함해 차이가 즉시 느껴지는지
3. 섹터 5 B22가 섹터 1~4보다 실제로 어렵게 느껴지는지
4. 섹터 6 B7의 네 단계가 각각 학습 가능하면서도 충분히 긴장되는지
5. 타이탄+순양함 12척 상태에서 회수 드론 때문에 사실상 무적이 되는 문제가 해소됐는지

특히 5번은 이번 아트 통합과 별개의 전투 경제·피해 안전망 밸런스 문제다. 다음 플레이테스트에서 재현 여부를 확인한 뒤 수치 조정을 별도 작업으로 진행하는 것이 안전하다.
