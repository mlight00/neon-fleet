## P1-1. banked 반복 적립 (강등 후 화력 무한 farming) — 해결

**원인**: 업그레이드 시 흡수 순양함 화력을 `squad.banked`에 누적하는데, 죽음 안전망(`onDronesDepleted`)의 등급 강등은 `tier`만 내리고 `banked`를 그대로 두었음 → 업글→강등→재업글 반복으로 banked 무한 적립.

**수정** (`js/entities.js`):
- 업그레이드별 증가분을 `this.bankStack`에 push (`checkEvolution`).
- 강등 시 `this.banked -= this.bankStack.pop()` 로 **정확히 롤백** (`onDronesDepleted`의 등급 강등 분기).

**검증** (실행 계측):
- 업그레이드 → `banked 1463`, 강등 → `banked 0` (정확 복원).
- 업글↔강등 3회 반복 시 banked peak가 **한 티어분(1463)에서 고정**, 누적되지 않음.

## P1-2. 위험(hazard) 노드 필터 여전히 깨짐 — 해결

**원인**: `buildEncounter`가 노드 시작부 N개 청크에 `isSafeChunk`(위협 없음)를 강제하는데, hazard 필터는 `debris/mine`(=위협)을 **요구**함 → 두 조건이 모순되어 `pickChunk`가 필터 만족 청크를 못 찾고 일반 청크로 폴백.

**수정** (`js/main.js`): hazard 노드는 안전-시작 청크 수를 **0**으로 (`safeCount = node.type === 'hazard' ? 0 : ...`). 이제 hazard 노드의 모든 청크가 순수 hazard 필터를 통과 → 항상 `debris/mine`.

**검증**: `tests/chunks.test.mjs`의 필터 보장 테스트(easy/mid/hard × 40회) 통과 — hazard 필터가 항상 debris/mine 청크를 반환.

## P1-3. 기존 잘못된 STAGE 기록 미마이그레이션 — 해결

**원인**: 이전 버그로 `save.stage`에 내부 난이도(`(sector-1)×6 + col + 1`, 첫 보스=6)가 저장됨. 기록 저장을 `r.sector`로 고쳤지만 **기존 저장값은 그대로 부풀려진 상태**.

**수정** (`js/save.js`): `createSave` 초기화 시 **1회 마이그레이션** — 원본 raw를 직접 파싱해 `stageMigrated` 플래그가 없으면 `stage = floor((stage-1)/6) + 1`로 환산 후 플래그 세팅. `DEFAULTS.stageMigrated = true`라 **신규 저장은 재환산되지 않음**(섹터 기록 보호). 멱등.

**검증** (`tests/save.test.mjs` 신규 2종):
- `[6→1, 12→2, 18→3, 1→1, 7→2]` 환산 + 다른 필드 보존 + 재생성 멱등.
- `stageMigrated:true`인 신규 저장은 stage 불변(섹터 3 유지).

## (추가) 플레이 중 발견 회귀 — 보스전 기함 종류 실종 — 해결

**원인**: 이전 리뷰의 "상단 HUD 겹침" 대응으로 보스전에 기함 상세줄을 통째로 숨겼더니 기함 종류를 알 수 없게 됨.

**수정** (`js/render.js`, `js/main.js`): 보스전에는 상단 **함대 줄(보스 HP바보다 위)** 에 짧은 기함 이름만 병기(`드론 N · 순양함 M · 기함 스트라이커`). 상세줄(트레잇·화력·게이지)은 계속 숨김 → 겹침 없이 기함 종류 노출.

**검증** (fillText 계측): 보스전 상단 줄 = `"드론 8기 · 순양함 0 · 기함 스트라이커"`, 보스명/HP와 별도 라인.

## 후속 리뷰 보완 (2차 지적 4건)

**① bankStack 자동 회귀 테스트 추가** — 은행 로직을 순수 함수(`bankUpgrade`/`bankDemote`, `logic.js`)로 추출해 Squad가 사용하도록 하고, 다음 테스트를 추가:
- 승급→강등 **10회 반복** 시 banked가 매 사이클 0으로 복귀하고 peak가 한 티어분에서 고정(누적 없음).
- 여러 티어 연속 승급 후 연속 강등 시 역순 정확 롤백해 0.
- 빈 스택 강등 안전(음수 방지).

**② README "60% 위력" 문구 수정** — `cruiserFireRatio`(0.60)는 실제 사격 계산에 쓰이지 않고 `cruiserPower=130`이 그대로 들어감. 문구를 "**드론 130기 상당 화력으로 함께 발사**"로 정정(`README.md`, `balance.js` 주석).

**③ 결과서 커밋** — 본 문서를 저장소에 커밋·푸시.

**④ 저장 마이그레이션 배포환경 E2E** — 실제 브라우저에서 구버전 localStorage 샘플(`{stage:6}`, 플래그 없음)로 현재 배포 코드(`save.js`)를 cache-bust import해 검증: **6→1, 12→2, 18→3, 7→2** 모두 정상 환산·영속화, `stageMigrated=true` 신규 저장은 불변(섹터 3 보존).
- **주의**: 업데이트 직후에도 STAGE 6이 남아 보이는 현상은 **브라우저 ES 모듈 캐시**(옛 `save.js` 재사용)가 원인 — 강력 새로고침 시 마이그레이션이 실행되어 해소됨. 코드 결함 아님.
