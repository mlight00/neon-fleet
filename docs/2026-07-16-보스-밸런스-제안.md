# 보스 밸런스 제안 (측정 근거 기반)

- 작성일: 2026-07-16
- 상태: **제안 — 미적용.** 사용자 승인 시 결함 수정과 **분리된 별도 커밋**으로 적용한다.
- 근거 측정: `docs/2026-07-16-업데이트-결함-수정-결과.md` §5

## 1. 현재 설정

```js
difficulty: { globalMult: 1.38, bossHpMult: 2.2, bossRateMult: 1.8 }
```

보스 최종 HP에는 `1.38 × 2.2 = 3.036배`가 적용되고, 발사 주기는 추가로 `1.8`로 나뉜다.

## 2. 측정 결과 요약

무강화, 각 섹터 대표 편대, 보스전만 계측.

| 섹터 | 화력 | 보스 총 HP | HP/화력 | 실측 DPS | 처치시간(완벽 회피) | 방치 시 |
|---|---|---|---|---|---|---|
| 1 | 200 | 16,364 | 82× | 247 | **66초** | 10.5초 전멸 |
| 3 | 1,500 | 420,411 | 280× | 1,778 | **237초** | 11.0초 전멸 |
| 6 | 5,520 | 2,066,424 | 374× | 13,495 | **153초** | 12.9초 전멸 |

## 3. 판정

### 잘 된 것 — 발사 강화(`bossRateMult 1.8`)는 유지 권장

세 섹터 모두 회피하지 않으면 10~13초에 전멸한다. 사용자가 지적한
**"보스를 그냥 멈춰서서 깨버렸다"는 완전히 해소**됐다.

직전까지 보스는 `scaleEnemy`를 거치지 않아 그동안의 난이도 상향에서 **발사 속도 버프를 한 번도 받지 못하고 있었다.**
"패턴이 쉽다"의 진짜 원인이 이것이었고, `bossRateMult`가 그 구멍을 메웠다.

### 과한 것 — 체력 배수(`bossHpMult 2.2`)는 과다

1. **전투가 너무 길다.** 완벽 회피 기준 66초 / **237초(4분)** / 153초.
   일반 노드가 수십 초인데 보스만 2~4분이면 계단이 지나치다.
2. **곡선이 비단조.** 섹터 3(237초) > 섹터 6(153초).
   고화력 구간은 `hpPerPowerCap`이 걸려 HP가 멈추는데 플레이어 DPS는 계속 올라서, **중반이 가장 지루하다.**
3. **무강화 섹터 1이 사실상 벽.** 66초 동안 피격 7회면 전멸 → 한 대도 안 맞아야 한다.
   신규 플레이어의 첫 보스로는 가혹하다.

핵심: 체력을 올려 얻는 건 **긴장이 아니라 시간**이다. 긴장은 이미 발사 강화가 만들고 있다.

## 4. 제안

### 4.1 1순위 — 체력 배수 완화 (한 줄)

```js
difficulty: { globalMult: 1.38, bossHpMult: 1.3, bossRateMult: 1.8 }
```

`bossHpMult` **2.2 → 1.3**. 발사 강화는 그대로 두어 "멈춰 있으면 죽는다"를 유지하면서 전투 길이만 줄인다.

예상 처치시간 (측정 DPS 기준, HP가 2.2→1.3으로 0.59배):

| 섹터 | 현재 | 제안 후(예상) |
|---|---|---|
| 1 | 66초 | **약 39초** |
| 3 | 237초 | **약 140초** |
| 6 | 153초 | **약 90초** |

### 4.2 2순위 — 중반 비단조 완화

섹터 3이 가장 오래 걸리는 원인은 `hpPerPowerCap`(40)이 저화력 구간에서 안 걸려
`mods.boss`(준지수)가 그대로 곱해지기 때문이다. 1순위 적용 후에도 섹터 3이 여전히 길면:

- `BAL.boss.hpPerPowerCap`을 40 → 28 부근으로 낮춰 중반 HP 상한을 앞당긴다
- 또는 `stageMods().boss`의 제곱항(`0.04·g²`)을 완화한다

**1순위만 먼저 적용해 실플레이로 확인한 뒤 판단할 것을 권한다.**

### 4.3 적용하지 말 것

- `globalMult 1.38` — 사용자가 "일반 적 난이도는 적절하다"고 평가. **손대지 않는다.**
- `bossRateMult 1.8` — "멈춰서 격파" 해결의 핵심. 낮추면 문제가 재발한다.

## 5. 적용 시 절차

1. `js/balance.js`의 `bossHpMult`만 수정 (한 줄)
2. `node --test` 전체 통과 확인 (밸런스 상수라 테스트 영향 없음이 예상됨)
3. 브라우저에서 섹터 1/3/6 처치시간 재측정 → 본 문서에 실측치 추가
4. 결함 수정 커밋과 **분리된 별도 커밋**으로 기록
