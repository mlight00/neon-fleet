## 0. Phase 2 핵심 원칙

1. 신규 조작 버튼을 추가하지 않는다.
2. 근접 회피는 위험을 감수한 플레이에 보상을 주되 필수 진행 조건으로 만들지 않는다.
3. 적탄 하나는 근접 회피 보상을 최대 한 번만 준다.
4. 무적·보스 연출·일시정지 중에는 근접 회피를 적립할 수 없다.
5. 키스톤은 원정당 정확히 하나만 선택한다.
6. 키스톤은 단순 수치 상승이 아니라 행동이나 공격 형태를 바꿔야 한다.
7. 키스톤의 장점과 대가를 카드에 함께 명시한다.
8. 신규 보스는 일반 공격만으로도 처치 가능해야 한다. FLOW와 차지는 그로기 보너스 수단이지 필수 열쇠가 아니다.
9. 특정 무기·교리·키스톤이 없으면 피해를 줄 수 없는 완전 면역은 금지한다.
10. 기존 B7~B15 보스 9종의 패턴·수치·로스터 순서를 불필요하게 변경하지 않는다.
11. 신규 이미지·사운드가 없어도 Canvas 도형과 기존 효과·SFX로 완전히 플레이 가능해야 한다.
12. 저장 스키마, 격납고, 승급·강등, `bankStack`, 저장 마이그레이션을 변경하지 않는다.
13. `entities.js`·`bosses.js` 추가 분할은 이번 범위에 포함하지 않는다.
14. 한 시스템을 구현한 뒤 테스트와 브라우저 스모크를 통과하고 다음 시스템으로 넘어간다.

## 1. 플레이 경험 목표

Phase 2의 전투 리듬은 다음과 같아야 한다.

```text
적탄을 멀리 피함
→ 안전하지만 추가 보상 없음

적탄을 아슬아슬하게 스침
→ FLOW 상승 + 콤보 피드백
→ FLOW 100에서 NEON RUSH 자동 발동
→ 짧은 화력·충전·기동 보너스

섹터 1 보스 격파
→ 키스톤 3종 중 1개 선택
→ 군체/랜스/위상 중심의 플레이 규칙 변화

후반 신규 보스 네온 아비터
→ 탄막의 안전 틈을 읽고 근접 회피하거나 3단 랜스로 압박
→ STAGGER 누적
→ 짧은 BREAK 구간에 집중 공격
```

완료 후 플레이어가 느껴야 할 차이:

- 단순히 화면 끝으로 도망가는 것보다 탄의 가장자리를 스치는 선택이 재미있다.
- 피격되면 FLOW 기회를 잃기 때문에 욕심과 안전 사이의 판단이 생긴다.
- 키스톤 하나가 원정의 전투 리듬을 명확하게 바꾼다.
- 신규 보스는 HP만 높은 보스가 아니라 회피·차지·집중 공격의 순환을 요구한다.
- 기존 조작을 전혀 배우지 않은 플레이어도 일반 이동과 자동사격만으로 진행할 수 있다.

## 3. 근접 회피 시스템 — FLOW / NEON RUSH

### 3.1 역할

적탄에 맞지 않으면서 피격 원의 바깥쪽 좁은 구간을 통과하면 근접 회피로 인정한다. 근접 회피는 FLOW를 쌓고, 최대치에서 자동으로 NEON RUSH를 발동한다.

### 3.2 밸런스 초기값

`BAL.flow`에 모든 수치를 둔다.

```js
flow: {
  grazeBand: 18,          // 실제 피격 경계 바깥 추가 판정 폭(px)
  gain: 10,               // 적탄 1개당 FLOW
  max: 100,
  decayDelay: 1.5,        // 마지막 graze 후 감소 시작
  decayPerSec: 8,
  minBulletAge: 0.12,     // 생성 직후 겹침 farming 방지
  rushDuration: 4.0,
  rushDamageMult: 1.18,
  rushChargeSpeedMult: 1.20,
  rushMoveResponseMult: 1.15,
  hitLoss: 35,            // RUSH 전 피격 시 FLOW 손실
  textCooldown: 0.12,     // GRAZE 문구·SFX 스팸 제한
},
```

초기 수치는 코드에 하드코딩하지 않는다.

### 3.3 Squad 원정 상태

`Squad` 생성자에 저장되지 않는 원정 내부 상태를 추가한다.

```js
this.flow = 0;
this.rushT = 0;
this.grazeCombo = 0;
this.lastGrazeT = -Infinity;
this.grazeFxT = 0;
```

필요하다면 키스톤 내부 상태는 `this.keystoneState` 한 객체 아래에 둔다.

### 3.4 순수 로직

`flow.js`는 최소 다음 순수 함수를 제공한다.

```js
export function isGrazeDistance(distance, hitRadius, bulletRadius, band) {}
export function addFlow(state, cfg) {}
export function updateFlow(state, dt, cfg) {}
export function onFlowHit(state, cfg) {}
```

권장 반환 형태:

```js
{
  flow,
  rushT,
  combo,
  sinceGraze,
  rushStarted,
  rushEnded,
}
```

렌더링·오디오·world 접근은 순수 함수에 넣지 않는다.

### 3.5 EnemyShot 판정 순서

`EnemyShot`에 다음 상태를 추가한다.

```js
this.age = 0;
this.grazed = false;
```

매 프레임 순서:

1. 탄 이동
2. 실제 피격 판정
3. 실제 피격이면 탄 제거 및 피해 처리, 같은 프레임 graze 금지
4. 피격하지 않았을 때만 graze 거리 판정
5. `age >= minBulletAge`, `!grazed`, `squad.invulnT <= 0`, 플레이 상태일 때만 적립
6. 인정 시 `grazed = true`

거리 정의:

```text
hitDistance = squad.hitRadius + bullet.r
graze 인정 = distance > hitDistance
          && distance <= hitDistance + grazeBand
```

경계값은 테스트로 고정한다.

### 3.6 farming 방지

- 적탄 하나당 1회만 적립한다.
- 관통하거나 정지한 탄도 같은 탄이면 다시 적립하지 않는다.
- 진화 무적, 보스 사망 연출, flythrough, pause, drafting 중에는 적립하지 않는다.
- 이미 `dead`인 탄은 적립하지 않는다.
- 플레이어에게 실제로 맞은 탄은 graze를 지급하지 않는다.
- 널 커터가 제거한 탄은 제거 순간 graze를 지급하지 않는다.
- 화면 밖 탄은 적립하지 않는다.

### 3.7 FLOW 감소와 피격

마지막 근접 회피 후 1.5초가 지나면 초당 8씩 감소한다.

전투 피해를 실제로 받았을 때:

- RUSH가 아니면 FLOW 35 감소
- RUSH 중이면 RUSH 즉시 종료, FLOW는 0
- 콤보 0으로 초기화

다음은 피격으로 보지 않는다.

- 좋은/나쁜 게이트 연산
- 정비 회복
- 무기·보상 획득
- 업그레이드 비용
- 승급·강등 내부 수치 변경
- 보호막 또는 무적으로 실제 드론 손실이 0인 공격

적탄 피해와 적/운석/기뢰 접촉 피해처럼 실제 전투 손실이 발생한 경우만 `onCombatHit()`을 호출한다.

### 3.8 NEON RUSH

FLOW가 100에 도달하면 자동 발동한다. 추가 버튼은 없다.

발동 효과:

```text
지속 4초
모든 아군 사격 피해 ×1.18
차지 속도 ×1.20
이동 목표 추적 반응 ×1.15
```

적용 대상:

- 기함 자동사격
- 호위 드론 사격
- 순양함 사격
- 무기 진화의 직접·파생 피해
- 차지 랜스의 충전 속도

중복 적용 금지:

- `phaseMul`, 모듈, 교리와 곱연산은 한 번씩만 적용한다.
- RUSH 피해 배수가 도탄·분열·폭발에서 다시 중복 곱해지지 않게 한다.
- RUSH 중 FLOW 추가 적립은 막는다. RUSH 종료 후 다시 적립 가능하다.

발동 연출:

- `NEON RUSH!` 텍스트
- 청록+자홍 외곽 링
- 함대 후광
- 기존 SFX 중 가장 가까운 효과 사용
- 화면 전체 플래시는 0.25 이하로 제한

### 3.9 HUD

모바일·데스크톱 모두 다음을 표시한다.

```text
FLOW 70
[███████---]
```

RUSH 중:

```text
NEON RUSH 3.2s
```

규칙:

- 기존 보스 HP, 함대 정보, 무기 정보와 겹치지 않는다.
- 모바일 390×844에서 한 줄을 넘지 않는다.
- FLOW 0일 때는 흐리게 표시하되 완전히 숨기지 않아 시스템 존재를 알 수 있게 한다.
- 색상만으로 상태를 전달하지 않고 `FLOW`, `RUSH` 텍스트를 함께 쓴다.

## 4. 키스톤 시스템 — 원정당 1개

### 4.1 획득 시점

첫 번째 섹터 보스 격파 후, 다음 섹터 맵으로 넘어가기 전에 키스톤 3장을 모두 보여준다.

```text
섹터 1 보스 격파
→ 보스 사망·통과 연출 완료
→ 키스톤 3택
→ 선택 완료
→ 섹터 2 맵
```

- 키스톤 선택과 일반 모듈 드래프트가 동시에 열리지 않는다.
- 일시정지·차지 버튼을 숨긴다.
- 선택 중 `input.charging = false`로 초기화한다.
- 새 원정에서는 다시 `null`로 시작한다.
- 강등·무기 교체·교리 유지와 무관하게 원정 끝까지 유지한다.
- localStorage에 저장하지 않는다.

### 4.2 데이터 구조

`keystones.js`에 정의한다.

```js
export const KEYSTONES = [
  // 아래 3종
];

export const KEYSTONE_BY_ID = ...;
export function keystoneEffects(id) {};
```

원정 상태:

```js
run.keystone = null;
```

또는 전투 접근성을 위해 `squad.keystone`으로 통일해도 되지만 두 곳에 중복 저장하지 않는다.

### 4.3 키스톤 1 — 군체 용광로 `swarm_forge`

카드:

```text
🐝 군체 용광로
10킬마다 8초간 유령 순양함 2척 전개
유령 순양함 활성 중 호위·순양함 사격 +25%
대가: 기함 직접 사격 −10%
```

구현 규칙:

- 실제 적 처치만 카운트한다. 크리스탈·수송선·장애물 파괴는 제외한다.
- 10킬 도달 시 `forgeT = 8`.
- 이미 활성 중 다시 10킬이면 8초를 더하되 최대 16초까지만 연장한다.
- 유령 순양함은 체력·충돌 판정이 없는 시각적 편대다.
- 기존 `fireSupport` 또는 `fireEscort` 경로에 효과를 한 번만 적용한다.
- 군체 교리가 없어도 작동하지만 군체 교리와 조합 시 가장 강하게 느껴져야 한다.
- 보스 소환물 처치도 킬로 인정하되 한 프레임 다중 처치가 중복 누락되지 않게 한다.

### 4.4 키스톤 2 — 공명 랜스 `lance_echo`

카드:

```text
⚡ 공명 랜스
3단 이상 차지 랜스가 0.35초 후 좁은 메아리 랜스를 재발사
메아리 피해 45%, 폭 65%
대가: 자동사격 피해 −12%
```

구현 규칙:

- 1·2단 랜스에는 발동하지 않는다.
- 원본 랜스 1회당 메아리 1회만 예약한다.
- 메아리가 다시 메아리를 만들 수 없다.
- 원본 발사 위치와 방향을 저장한다.
- 0.35초 뒤 해당 세로 컬럼에 원본 최종 피해의 45%를 적용한다.
- 폭은 원본의 65%다.
- 메아리도 관통하지만 적탄 제거는 하지 않는다.
- 원본이 랜스 강습 3단 관통 조건을 만족했다면 메아리도 같은 `pierceDefense`를 계승한다.
- 보스 사망 연출·노드 종료·원정 종료 시 예약된 메아리를 정리한다.

### 4.5 키스톤 3 — 위상 잔상 `phase_afterimage`

카드:

```text
◈ 위상 잔상
근접 회피 3회마다 함대 주변 70px 위상 파동
파동이 적탄을 최대 8발 제거
대가: 피격 시 FLOW와 RUSH를 전부 잃음
```

구현 규칙:

- graze 이벤트 3회마다 한 번 발동한다.
- 반경 70px 안의 살아 있는 적탄을 거리순 최대 8발 제거한다.
- 제거된 탄은 graze를 추가 지급하지 않는다.
- 같은 graze 이벤트가 FLOW 증가와 잔상 카운트에 정확히 한 번씩만 반영된다.
- 보스전·일반전 모두 작동한다.
- 적탄이 없을 때도 카운트는 소비하고 파동 연출은 보여준다.
- 피격 시 기본 FLOW 규칙 대신 FLOW 0, RUSH 즉시 종료를 적용한다.

### 4.6 키스톤 UI

전용 `showKeystoneDraft()`를 구현한다.

각 카드에 반드시 표시:

- 아이콘
- 이름
- 행동 변화
- 장점
- `대가:` 문구
- `원정당 1개` 안내

HUD에는 선택한 키스톤 아이콘 하나만 표시한다. 기존 모듈 요약 줄이 길어지지 않게 별도 짧은 위치를 사용한다.

### 4.7 키스톤 중립성

키스톤 미선택 상태에서는 모든 배수가 1, 카운터 0, 예약 없음이어야 한다. Phase 2 추가 전과 전투 결과가 동일해야 한다.

## 5. 신규 상호작용형 보스 — B22 네온 아비터

### 5.1 배경과 로스터

기존 B16~B21은 일반 적 슬롯이므로 신규 보스 ID는 `B22`를 사용한다.

```js
{ id: 'B22', name: 'NEON ARBITER', korName: '네온 아비터' }
```

- `BOSS_ROSTER` 뒤에 추가한다.
- `SPRITE_SIZES.B22 = 150`.
- `RASTER_ART.C.B22` 슬롯을 등록하되 PNG가 없어도 404 콘솔 경고를 내지 않고 Canvas 폴백으로 정상 플레이되어야 한다.
- 기존 B7 하이브 퀸 이미지 복사만으로 표시하지 않는다. B22 전용 Canvas 폴백 외형을 만든다.

### 5.2 역할

네온 아비터는 탄막의 안전 틈을 읽어 graze를 만들거나 3단 랜스로 압박하면 STAGGER가 쌓이는 보스다. STAGGER가 최대가 되면 짧게 BREAK되어 공격을 멈추고 추가 피해를 받는다.

일반 자동사격도 항상 100% 기본 피해를 준다. STAGGER는 처치 시간을 단축하는 보너스다.

### 5.3 밸런스 초기값

`BAL.bossPatterns.B22` 또는 별도 `BAL.neonArbiter` 블록에 둔다.

```js
B22: {
  kind: 'arbiter',
  staggerMax: 10,
  grazeStagger: 1,
  lanceStagger: 2,
  breakDuration: 1.6,
  breakDamageMult: 1.25,
  staggerCooldown: 2.0,
  wallInterval: 1.35,
  wallTelegraph: 0.65,
  wallCount: 8,
  wallGapSlots: 2,
  wallSpeed: 170,
  ringInterval: 1.55,
  ringCount: 14,
  ringGapDeg: 55,
  ringSpeed: 155,
  enrageIntervalMult: 0.82,
}
```

실플레이 후 ±15% 범위에서 조정할 수 있지만 결과서에 변경 근거를 적는다.

### 5.4 보스 상태

```js
this.arbiterPhase = 1;
this.stagger = 0;
this.breakT = 0;
this.staggerCooldownT = 0;
this.warning = null;
this.signatureToggle = 0;
```

HP 단계:

```text
100%~66%: GAP WALL 중심
66%~33%: BROKEN RING 중심
33%~0%: 두 패턴 교대, 간격 ×0.82
```

단계 변경 시 텍스트와 링 연출을 한 번만 표시한다.

### 5.5 패턴 A — GAP WALL

화면 폭을 8슬롯으로 나눈 뒤 적탄 벽을 만든다. 연속한 2슬롯은 비워 안전 통로로 남긴다.

순서:

1. 안전 통로 위치 결정
2. 0.65초 동안 위험 슬롯에 세로 경고선 표시
3. 경고 종료 후 위험 슬롯에서 아래 방향 탄 발사
4. 안전 통로는 편대 실제 폭과 `hitRadius`를 고려해 최소 72px 이상 보장

규칙:

- 안전 통로가 화면 밖으로 잘리지 않는다.
- 직전 안전 통로와 같은 위치가 3회 연속 나오지 않는다.
- `world.rng`를 사용해 테스트 재현성을 보장한다.
- 경고 중에는 해당 패턴 탄을 미리 발사하지 않는다.

### 5.6 패턴 B — BROKEN RING

14발 원형탄에서 55도 이상의 빈 각도를 하나 남긴 고리 탄막이다.

- 빈 각도 중심은 편대 방향에서 좌우 35도 범위로 정한다.
- 플레이어가 읽고 통과할 시간이 최소 1.2초 있어야 한다.
- 링 전체가 한 프레임에 샷캡 때문에 절반만 생성되지 않도록 발사 전 가용 탄 수를 확인한다.
- 샷캡이 부족하면 탄 수를 줄이되 빈 각도는 유지한다.

### 5.7 STAGGER 획득

네온 아비터가 살아 있고 BREAK·쿨다운 상태가 아닐 때:

```text
보스전에서 적탄 graze 1회 → STAGGER +1
3단 이상 원본 차지 랜스 직격 → STAGGER +2
메아리 랜스 → STAGGER 추가 없음
일반 자동사격 → STAGGER 없음
```

중복 방지:

- 적탄 하나의 graze는 이미 FLOW와 마찬가지로 1회다.
- 하나의 랜스가 다중 보스나 다중 프레임에서 같은 B22에 여러 번 STAGGER를 주지 않는다.
- 보스가 여러 기면 B22 자신에게 관련된 graze만 처리하기 어렵기 때문에, Phase 2에서는 B22 등장 시 해당 보스 슬롯을 단독 보스로 강제하거나 STAGGER 이벤트 대상을 명시적으로 결정한다.

권장안: **B22는 항상 단독 보스**로 스폰한다. 기존 다중 보스 계산에서 B22가 포함되면 `bossN = 1`로 제한한다.

### 5.8 BREAK

STAGGER 10 도달 시:

- `breakT = 1.6`
- STAGGER 0으로 초기화
- 모든 공격 타이머 정지
- 이동 흔들림 정지
- 받는 피해 ×1.25
- `BREAK!` 텍스트와 흰색 균열 연출
- BREAK 종료 후 STAGGER 적립 2초 쿨다운

BREAK 중에도 보스는 피해를 받고 사망할 수 있다.

### 5.9 피격 문맥

`Boss.hitByBullet(dmg, world, ctx = null)` 계약을 사용한다.

랜스 문맥에 최소 다음을 포함한다.

```js
{
  lance: true,
  stage,
  pierceDefense,
  echo: false,
  attackId,
}
```

메아리 랜스는:

```js
{ ...originalCtx, echo: true }
```

`attackId`는 한 공격의 STAGGER 중복 적립 방지에 사용한다.

일반 탄환 객체에는 임의의 `lance` 필드를 추가하지 않는다.

### 5.10 Canvas 폴백 외형

PNG 없이도 다음 특징이 보여야 한다.

- 중앙의 흰색 판결 코어
- 좌우로 회전하는 청록·자홍 저울 팔
- STAGGER가 높을수록 코어 균열 증가
- BREAK 중 팔이 멈추고 흰색 점멸
- 단계 3에서 외곽 링이 붉은색으로 변화

색상만으로 구분하지 않고 `STAGGER`, `BREAK` HUD 텍스트를 함께 표시한다.

### 5.11 보스 HUD

B22 보스 HP 아래에 짧은 보조 바를 추가한다.

```text
STAGGER 6/10
```

BREAK 중:

```text
BREAK 1.2s · DAMAGE ×1.25
```

다른 보스에는 이 보조 바가 나타나지 않는다.

## 6. 시스템 연결 규칙

### 6.1 이벤트 흐름

권장 이벤트 흐름:

```text
EnemyShot graze 판정
→ squad.onGraze(world, shot)
→ FLOW 순수 로직 적용
→ keystone onGraze 훅
→ B22 onPlayerGraze 훅
→ 연출
```

각 단계는 같은 graze를 한 번만 소비한다.

### 6.2 킬 이벤트

기존 `onEnemyKilled(e, world)`에 키스톤 훅을 연결한다.

- 이미 죽은 적의 중복 호출 금지
- 크리스탈·수송선 제외
- 보스는 일반 적 10킬 카운터에 포함하지 않는다.
- 보스 소환물은 포함한다.

### 6.3 피해 배수 순서

자동사격 최종 계수의 권장 순서:

```text
기본 화력
× 무기 계수
× 무기 레벨
× 일반 모듈
× 교리
× RUSH
× 키스톤 장점/대가
```

파생탄은 원본에 이미 적용된 최종 피해를 비율로 나눈다. 파생탄에서 RUSH·키스톤 배수를 다시 곱하지 않는다.

보스 받는 피해:

```text
공격 최종 피해
× bossDmgMult
× B22 BREAK 배수
```

### 6.4 원정·노드 전환 정리

노드 진입/종료 시 정리:

- 예약된 메아리 랜스
- 임시 유령 순양함 타이머
- B22 경고·BREAK·STAGGER 상태
- 적탄의 graze 상태는 탄과 함께 제거

원정 동안 유지:

- 선택한 키스톤 id
- FLOW 값은 노드 종료 시 0으로 초기화
- 키스톤 영구 선택
- 키스톤 고유 누적 카운터는 노드 시작 시 0으로 초기화

RUSH를 맵 화면으로 가져가지 않는다.

## 7. 성능·안전 기준

1. graze 판정은 기존 EnemyShot 업데이트의 거리 계산 1회로 끝낸다.
2. 별도 전역 이중 루프로 모든 탄×모든 개체를 매 프레임 검사하지 않는다.
3. 위상 잔상 탄 제거는 발동 순간에만 실행한다.
4. 적탄 샷캡을 우회하지 않는다.
5. B22 패턴은 샷캡 부족 시 안전 통로를 유지한 채 탄 수를 줄인다.
6. 메아리 랜스 예약 배열은 최대 3개로 제한한다.
7. 유령 순양함은 실제 개체 배열에 수십 개를 추가하지 않는다.
8. 60fps 기준 10분 헤드리스 시뮬레이션에서 배열이 지속 증가하지 않아야 한다.
9. 모바일에서 HUD 추가로 캔버스 유효 높이가 줄어들지 않게 한다.

## 8. 필수 자동 테스트

기존 테스트 전부를 유지하고 다음 테스트를 추가한다. 계산식 복사 테스트가 아니라 순수 함수와 실제 클래스를 적절히 함께 검증한다.

### 8.1 FLOW 순수 테스트

1. 실제 피격 경계 내부는 graze가 아니다.
2. 실제 피격 경계 바로 바깥은 graze다.
3. grazeBand 바깥은 graze가 아니다.
4. FLOW가 정확히 100에서 RUSH를 시작한다.
5. RUSH 시작 시 FLOW가 0으로 초기화된다.
6. 마지막 graze 후 1.5초 전에는 감소하지 않는다.
7. 감소 시작 후 초당 8씩 감소하고 0 아래로 내려가지 않는다.
8. 일반 피격 시 FLOW 35 감소한다.
9. RUSH 중 피격 시 RUSH가 즉시 끝난다.

### 8.2 EnemyShot 실제 클래스 테스트

1. 근접 통과한 탄이 FLOW를 한 번만 지급한다.
2. 같은 탄이 여러 프레임 근접해도 중복 지급하지 않는다.
3. 실제 명중한 탄은 graze를 지급하지 않는다.
4. 생성 0.12초 이전 탄은 graze를 지급하지 않는다.
5. 진화 무적 중에는 graze를 지급하지 않는다.
6. dead 탄과 널 커터로 제거된 탄은 지급하지 않는다.

### 8.3 RUSH 배수 테스트

1. 키스톤·RUSH 없음은 기존 피해와 동일하다.
2. RUSH가 기함·호위·순양함 피해에 각각 한 번만 적용된다.
3. 도탄·분열·폭발에 RUSH가 이중 적용되지 않는다.
4. 차지 속도 ×1.20이 적용된다.
5. 위상 교리와 RUSH가 의도한 곱연산 한 번씩만 적용된다.

### 8.4 키스톤 테스트

1. 키스톤 정의는 정확히 3종이고 id가 유일하다.
2. 한 원정에서 한 번만 선택 가능하다.
3. 새 원정에서 `null`로 초기화된다.
4. 군체 용광로는 정확히 10킬에서 발동한다.
5. 비적대 보상 개체 파괴는 킬로 세지 않는다.
6. 군체 용광로 활성 시간은 최대 16초다.
7. 공명 랜스는 1·2단에서 발동하지 않는다.
8. 공명 랜스는 3단 이상 원본 1회당 메아리 1회만 예약한다.
9. 메아리는 재귀적으로 메아리를 만들지 않는다.
10. 메아리 피해 45%, 폭 65%가 정확하다.
11. 위상 잔상은 3번째 graze마다 발동한다.
12. 위상 잔상은 반경 내 거리순 최대 8발만 제거한다.
13. 위상 잔상이 제거한 탄은 FLOW를 지급하지 않는다.
14. 키스톤 미선택 상태는 모든 효과가 중립이다.

### 8.5 B22 테스트

1. `bossDefFor()` 순환에 B22가 포함된다.
2. B22는 `kind:'arbiter'` 패턴을 가진다.
3. B22 스폰 시 단독 보스로 제한된다.
4. HP 비율에 따라 단계 1→2→3이 정확히 한 번씩 전환된다.
5. GAP WALL은 항상 연속 안전 슬롯 2개를 남긴다.
6. 안전 통로 폭이 최소 72px다.
7. BROKEN RING은 최소 55도 빈 각도를 남긴다.
8. 동일 RNG 시 패턴이 재현된다.
9. graze 1회는 STAGGER +1이다.
10. 일반 자동사격은 STAGGER를 올리지 않는다.
11. 3단 이상 원본 랜스는 +2다.
12. 메아리 랜스는 STAGGER를 올리지 않는다.
13. 동일 `attackId`는 중복 STAGGER를 주지 않는다.
14. STAGGER 10에서 BREAK 1.6초가 시작된다.
15. BREAK 중 공격 타이머가 진행되지 않는다.
16. BREAK 중 받는 피해가 ×1.25다.
17. BREAK 종료 후 2초 쿨다운 동안 STAGGER가 쌓이지 않는다.
18. 일반 사격만으로도 B22를 처치할 수 있다.

### 8.6 회귀 테스트

다음 기존 불변식을 계속 통과해야 한다.

- banked farming 차단
- 저장 마이그레이션 멱등성
- hazard 노드 필터
- 강등 안전망
- 스캐빈저 32→48 보상
- 패러사이트 일반 65% / 랜스 관통 100%
- 무기 진화 6종
- 교리 3종
- Boss 재export 동일성

신규 테스트는 최소 35개 이상 추가한다. 단순 개수 채우기용 중복 테스트는 금지한다.

4. RUSH 중 피격 → 즉시 종료
5. 섹터 1 보스 격파 → 키스톤 3택 정확히 한 번
6. 군체 용광로 → 10킬 후 유령 순양함·타이머 확인
7. 공명 랜스 → 3단 원본 뒤 0.35초 메아리 확인
8. 위상 잔상 → 3번째 graze에서 최대 8발 제거 확인
9. B22 GAP WALL의 경고와 안전 통로 확인
10. B22 BROKEN RING 빈 각도 통과 가능 확인
11. graze와 원본 3단 랜스로 STAGGER 누적 확인
12. BREAK 1.6초 동안 공격 정지·받는 피해 증가 확인
13. FLOW를 전혀 사용하지 않고 자동사격만으로 B22 처치 가능 확인
14. 보스 격파 후 맵 복귀·다음 섹터·키스톤 유지 확인
15. 사망 후 새 원정에서 FLOW·키스톤 초기화 확인

각 키스톤은 최소 한 번의 실제 트랙과 한 번의 보스전에서 검증한다.
